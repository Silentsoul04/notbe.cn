阿里云全面支持SMB协议服务，为数据访问赋能

Jan 15, 2018

阿里云NAS近日宣布，SMB（Server Message Block）文件系统访问协议服务正式商业化， 相比之前的公测版本，商业化后的SMB文件系统服务性能大幅提升，可同时支持容量型和性能型NAS文件系统，读写性能最大提升3倍，元数据操作（创建删除文件等）最大提升20倍。

目前，阿里云是国际上唯一一家同时支持NFS和SMB协议的主流公有云厂商，在产品性能方面处于国际领先地位，价格却更加优惠。

服务Web内容发布和管理
Windows Server是非常流行的网站建构平台。目前，全球40%左右的站点Web Server采用微软IIS (来自Netcraft February 2017 Web Server Survey)，很多网站和博客系统也是基于WordPress、Joomla等内容管理系统，在Windows IIS平台实现的。

很多用户选择使用阿里云独享型Windows ECS来提供网站服务，通过将网站内容资源集中存储阿里云NAS文件系统上，IIS可以和访问本地文件系统一样访问阿里云NAS上的数据，从而帮助用户网站服务实现存储和计算分离，同时计算资源和存储资源能够按需弹性扩展, 通过阿里云负载均衡服务搭建一个弹性容错的网站架构。如图所示：

现有的Windows应用上云
很多用户在Windows平台上有大量程序和数据的历史积淀，特别是很多传统的Windows程序如企业管理信息系统，ERP和CRM系统等往往具有很长的软件生命周期。这类应用在运行时往往依赖于大规模的本地文件系统或者on-premises的SAN或者NAS，这类应用上云往往是用户的业务全面上云面临的难题。

通过阿里云NAS的SMB文件系统，这类Windows应用可以不需修改直接上云，保证高性能数据访问和高可靠数据存储。例如，医疗行业中专用医学影像存档与通信系统（Picture archiving and communication system，PACS）使用on-premises的NAS来存储医学影像，医生通过办公室或者诊室的Windows客户端从NAS来提取和分析这些影像。阿里云NAS在支持SMB后这样的系统可直接上云。

企业级文件共享和虚拟桌面服务
随着越来越多的企业上云，提供高性能高可用的文件共享服务来替代传统企业中的文件服务器的需求日趋紧迫。当前Windows环境的虚拟桌面份额很大，支持SMB协议访问NAS至关重要，一些典型的应用场景如下：
• 企业文件共享服务，例如文档发布；
• RemoteApp发布和管理；
• 个人桌面配置ECS间共享和自由迁移；
• 管理员可以通过阿里云NAS创建企业级或部门级的共享文件系统，不同的用户在该文件系统中拥有个人目录，支持通过不同的虚拟桌面访问和共享数据程序；
• Windows Hyper-V直接挂载磁盘镜像文件(.VHD)。
下图显示了Windows环境虚拟桌面通过阿里云NAS共享数据的场景。

让媒体信息处理更便捷
目前，很多视频音频应用（媒体非编制作，动画渲染）需要在Windows和Mac OS X平台运行，阿里云NAS对SMB的支持使这些应用更能直接上云。用户可以按需部署阿里云ECS，批量处理阿里云NAS上的音视频数据。

数据备份和恢复
Windows发行版自带的备份程序如Windows Server 2008 和2012中的Windows Server Backup，用户可以按需配置周期性的自动目录备份，并将备份的内容保存到NAS。
