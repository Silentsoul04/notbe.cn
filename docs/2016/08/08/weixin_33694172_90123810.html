安装Autodesk Vault Professional 2014 Server （ADMS）

Aug 8, 2016


作者： 杜长宇 Autodesk ADN开发顾问

 

工程设计或机械设计的图档越来越多，对于一个严肃的设计单位来说，非常有必要对所有的图档进行集中管理并进行追踪。Autodesk® Vault 数据管理软件提供一套全面且功能强大的工具及功能，以帮助工程师、设计师、制造商、建筑师及施工团队提高组织、管理及追踪其设计数据的效率。 这篇文章介绍一下Vault Professional服务器的安装部署。对于Vault的部署，还是要一点技术含量的，如果你是很大的设计单位，工程人员分布在各地，那么你可能需要设置多站点，如果有很多设计人员频繁的访问vault，那么也会面临服务器的性能调整问题。

安装Vault Professional Server需要下面几个步骤

  1. 安装前的准备和规划
  2. 安装License Manager并配置授权服务器
  3. 安装配置SQL Server， IIS
  4. 安装Vault Server（ADMS）
  5. 安装Vault Client

 

安装前的准备和规划

对于Autodesk vault的配置部署，可以很简单到把所有的东西都安装在一台计算机上，也可以根据实际情况分开部署，实施过程越复杂，需要考虑的信息就越多。需要考虑的因素包括用用户数，服务器数，服务器硬件规格，服务器操作系统，访问频率等，具体的可以参考这里。如果你不清楚该用什么的硬件配置能满足需要，IBM Sizing Guide 应该对你有帮助。

安装License Manager并配置授权服务器

Autodesk Vault Professional需要网络授权，所以需要首先配置授权服务器。启动Vault Server的安装程序，首先同Utilites and Tool里面找到Network License Manager并安装。你需要向Autodesk申请授权文件（许可文件），http://register.autodesk.com，你需要提供你的授权服务器的机器名和网卡的Mac地址，Autodesk授权管理部门会发给你一段文本，拷贝到记事本中保存了*.lic文件备用。 Network License Manger启动后如下图，你可以参考峻祁连以前的文章和安装程序带的pdf文档来设置授权服务器。在此略过。

还可以参看这里http://wikihelp.autodesk.com/Vault/chs/Help/Help/0298-dchedfhd298/0299-dchedfhd299/0331-dchedjdd331/0332-dchedjdf332

安装配置SQL Server， IIS

Autodesk vault使用SQL Server存储文件的元数据，并通过Web Service提供服务，所以需要安装SQL Server 和IIS。如果你不需要单独配置SQL Server，那么可以让Vault Server安装程序帮你自动安装一个SQL Server Express版，只是数据量、连接数方面有一点限制，参看上面的规划部分。如果你是自己先安装了SQL SERVER 2008版，注意实例名要是AUTODESKVAULT，sa用户的密码设置为Vault的确实密码AutodeskVault@26200。 关于SQL Server的安装配置可以参考峻祁连的文章 ：图解安装Autodesk Vault Professional 2013 -- 安装SQL SERVER及其SQL SERVER远程访问的设置

安装Vault Server（ADMS）2014

下面开始安装Vault Server，看图少说话。安装之前看看看左下角的文档，心中有数。

如果你实在没事干，找个律师给他打电话，把下面的许可协议念给他听看能不能接受。如果没问题就点接受并继续：）

输入你的序列号和product Key, Vault Professional 2014的product Key是569F1，另外输入你在第2步配置好的授权服务器。

这里我之前已经安装好了SQL SERVER并把实例名设置为Autodeskvault,如果你没安装过，Vault安装程序会帮你安装一个SQL express。

往下拉一点是SQL SERVER的用户名和密码设置，我使用Vault的确实的设置，即sa，缺省密码AutodeskVault@26200
