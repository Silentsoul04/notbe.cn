关于linux qq的两个bug

Aug 17, 2017

   前天我看到ubuntu11.10版本发布了，很是激动。急忙就先下载了，制作了u盘启动盘。然后给自己的电脑安装上了。

      不过今天安装完qq，就发现了一些bug，根据先前的测试经验来看，能发现bug的地方，一般都还容易发现bug。
      呵呵，后面有bug发现的话，继续爆料。
      第一个bug是qq的版本问题，我在腾讯官网上下载的是linuxqq_v1.0.2-beta1_i386.deb，看描述是支持ubuntu 11.10的。不过安装过程中发现，它的版本定义错了。根据提示来看，qq的版本号没有以数字开头。解压deb包，找到DEBIAN目录，底下有个control文件，在里面第七行果然是Version: v1.0.2-beta1 。版本号以v开头，没有以数字开头。将这个修改之后。使用dkpg -b 命令重新制作安装包。然后使用dkpg来安装qq或者使用ubuntu software center来安装。
      关于第一个bug，我觉着有点奇怪，印象中先前在ubuntu 10.04中安装qq没碰到这个问题的。
      第二个bug是qq启动之后，屏幕就被卡住了，所有操作都做不了。先前在ubuntu10.10上安装了一样的qq版本之后，还能行，不过现在就很悲剧。可能是11.10采用了unity的原因吧。每次打开一个程序，虽然没有最大化，但总是要在屏幕最上面那一栏显示当前最上面的那个程序的菜单，而qq，估计是没这个菜单的。所以会卡住。
      先前ubuntu10.10采用gnome 2桌面系统时，没有这个问题。
      继续保持关注。
