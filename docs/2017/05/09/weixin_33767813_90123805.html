Autodesk Infrastructure Map Server(AIMS) 2012 常见错误

May 9, 2017


这里是安装AIMS2012中常见错误的解决办法，如果哪天不幸碰到了，希望能帮到你。关于AIMS 2012安装更详细的图文介绍可以看我的另外一篇文章图解安装Autodesk Infrastructure Map Server(AIMS) 2012。

 

当然，首先先看看Autodesk Infrastructure Map Server 2012系统需求，如果系统需求不满足，出现这样那样的问题也在所难免。

 

问题： Authentication Failed, Please enter a valid username and password.

 

可能的问题：

  1 用户名密码错误。 别以为不可能，有时候你就是敲错了而你没意识到。还有可能别的同事帮你改了没告诉你，我真遇到过有人是这样，呵呵。

缺省的用户名是 Administrator，注意A要大写，密码是admin全小写。

 

2. 你在安装AIMS 之前，IIS的配置是否有问题？ IIS+.net 安装，至少应该安装下图所示的IIS组件

 

3. 检查注册表中关于php的设置，如果没有的话，新建注册表项HKEY_LOCAL_MACHINE\SOFTWARE\PHP\5.3.3\IniFilePath 到 C:\Program Files\Autodesk\Autodesk Infrastructure Web Server Extension 2012\Php

 

4.还有一种可能性就是过了试用期. 进到MapGuide Server的log文件中看看出处日志，

C:\Program Files\Autodesk\Autodesk Infrastructure Map Server 2012\Logs\Error.log 文件，是否有 Invalid License等字样，如果有这个错误就是授权出错了， 需要运行ServerActivator.exe 激活，这个程序在C:\Program Files\Autodesk\Autodesk Infrastructure Map Server 2012\Bin 里。

 

如果用的是试用版，应该也需要运行这个"C:\Program Files\Autodesk\Autodesk Infrastructure Map Server 2012\Bin\ServerActivator.exe"工具来选择“try”试用。
