AutoCAD Map 3D 2010 Update 2 – released!

Feb 8, 2017


In our continuing effort to improve quality, performance and respond to our customers needs, we have released AutoCAD Map 3D 2010 Update 2.

Downloads

§ AutoCAD Map 3D 2010 Update 2

Changes Included in this Update

AutoCAD Map 3D 2010 Update 2 includes the AutoCAD 2010 Update 2 and also resolves the following Map 3D 2010 related issues:

General

· 

Cache files are left in the MapCache directory after closing AutoCAD Map 3D without saving the new drawing.

64-bit

· Saving a drawing with link templates sometimes crashes AutoCAD Map 3D.

ADE Save Back

· After an autosave, closing a drawing containing ADE queried objects crashes AutoCAD Map 3D.

Bulk Copy

· Bulk copying from a SHP data set to Oracle would sometimes fail to copy numeric fields.

CAD on GIS

· Checking out a large number of features crashes AutoCAD Map 3D.

· Attempting to join a linestring to a multi-linestring via the PEDIT command fails.

Classification

· Some menu items in the “Define Object Classification” dialog box are drawn covered in black.

COGO

· The MAPCGLIST command does not work on arcs.

Display Manager

· Loading a .layer file created by Topobase and then closing the drawing crashes AutoCAD Map 3D.

· Running the REGEN command after undoing the delete of a topology element crashes AutoCAD Map 3D.

· After creating a new map and closing the drawing, attempting to create and close another drawing crashes AutoCAD Map 3D.

· Drawing layers based on an XREF data are incorrectly drawn using the line weight from an FDO layer.

· Feature label performance is much slower after applying AutoCAD Map 3D 2010 Update 1.

· FDO raster layers containing elevation data do not respect scale ranges.

DWF Publishing

· Individual point features are merged into groups when publishing to DWF.

FDO Oracle

· Features containing null Z values cannot be edited.

· Remove version operation fails on Oracle 11.1.0.7.

· FDO curve strings containing only one segment are translated into SDO geometry with the wrong SDO_ETYPE definition.

· SQL requests are incorrectly constructed when table names have a length of 30 characters.

FDO Raster

· FDO rasters draw at the wrong location in layout view if the layer coordinate system is different from the map coordinate system.

· FDO rasters plot at the wrong location if the layer coordinate system is different from the map coordinate system.

· Adding a bitonal raster image crashes AutoCAD Map 3D.

· Gaps sometimes display between tiled images.

· Themed DEMs are not correctly displayed when running AutoCAD Map 3D on non-English operating systems.

· After connecting to a raster image with a very long file name, clicking on the connection in the Data Connect dialog crashes AutoCAD Map 3D.

· Zooming in/out after the raster file is renamed crashes AutoCAD Map 3D.

FDO SHP

· Attributes containing Hebrew, Greek, or Cyrillic characters are not displayed correctly in the Data Table.

· Under certain conditions, a SHP file can become corrupted when updating the geometry of features.

· Connecting to a SHP data set containing an invalid SHX index file crashes AutoCAD Map 3D.

FDO SQL Server Spatial

· Cannot set identity properties for view-based classes in which the view is based on multiple tables.

FDO WFS

· AutoCAD Map 3D cannot consume WFS layers published by MapGuide Enterprise 2008/2009.

FDO WMS

· Enabling the “Cache WMS data” option in the Data Connect dialog after connecting to a WMS service crashes Map.

· Connecting to a WMS service fails with an unhandled exception while trying to read the WMS schema.

GIS Joins

· Selecting a large number of features on layer containing joined data uses excessive memory and can crash AutoCAD Map 3D.

Globalization

· AutoCAD Map 3D sometimes crashes in module AcMapRightClickMenu.arx during start up.

Image

· Inserting an image in a network folder using the MAPIINSERT command crashes AutoCAD Map 3D.

· The MAPIMANAGE command crashes AutoCAD Map 3D on a system configured with the Windows /3GB startup switch.

Import/Export

· Memory is not reclaimed after running the MAPEXPORTCURRENTMAPTODWG command.

· Object data does not get exported when running the _MAPEXPORT command from the CLI.

· EPF files are not honored when running the _MAPEXPORT command from the CLI.

· The MAPIMPORTFDO and MAPEXPORTFDO commands do not list existing ArcSDE connections.

· Mapping multiple layers to the same DGN level when exporting to DGN does not function correctly.

Install/Uninstall

· An error message is displayed when installing AutoCAD Map 3D 2010 Update 1 on a non-English version of AutoCAD Map 3D.

MapGuide Interoperability

· Using Publish to MapGuide on a map containing FDO layers styled using external symbol libraries can crash AutoCAD Map 3D.

.NET API

· MPolygon objects cannot be created via the AcMPolygonMGD .NET API.

Plot/Publish

· Advanced placement feature labels do not display when plotting to a DWF or PDF.

Schema Editor

· With database connections, the Schema Editor displays an incomplete schema after adding new properties.

Topology

· The Attribute option of the MAPTHEMTOPO command doesn't work when running from the CLI.

Transform/Rubbersheet

· The ADERSHEET command mishandles MultiLeader and MLine entities.

UI

· Closing an inactive drawing sometimes crashes AutoCAD Map 3D.

· Closing a drawing while a nested object class node is expanded and selected in the Map Explorer panel crashes AutoCAD Map 3D.

· Invoking buttons on the Feature Class Contextual ribbon tab when a feature class is not selected crashes AutoCAD Map 3D.

Workflow

· Custom context menus added using the ActivityDesignerVerb API display as empty strings.

· Custom titles set on the File Open/Save dialog via the FileUIEditorParametersAttribute API display as empty strings.

Please check Readme for installation instructions and other Update 2 details.

Our sincere thanks to our Customer Response Team and everybody else who got involved with this important Update release.
