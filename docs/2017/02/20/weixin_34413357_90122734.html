二、调整 GRUB 进入操作系统菜单等待时间

1、GRUB：是linux下用来引导不同的操作系统的一个管理器，打开 /boot/grup/grup.conf，timeout 就是等待时间，单位是秒

2、GRUB 配置文件详解：vim /boot/grup/grup.conf

# grub.conf generated by anaconda
#
# Note that you do not have to rerun grub after making changes to this file
# NOTICE:  You do not have a /boot partition.  This means that
#          all kernel and initrd paths are relative to /, eg.
#          root (hd1,8)
#          kernel /boot/vmlinuz-version ro root=/dev/sdb9
#          initrd /boot/initrd-[generic-]version.img
#boot=/dev/sda
default=0
timeout=5
splashimage=(hd1,8)/boot/grub/splash.xpm.gz
hiddenmenu
title Demon (2.6.32-431.el6.x86_64)
root (hd1,8)
kernel /boot/vmlinuz-2.6.32-431.el6.x86_64 ro root=UUID=8b170155-dfeb-4b5d-be79-46ef7a605e19
rd_NO_LUKS  KEYBOARDTYPE=pc KEYTABLE=us rd_NO_MD crashkernel=128MLANG=zh_CN.UTF-8
rd_NO_LVM rd_NO_DM rhgb quiet
initrd /boot/initramfs-2.6.32-431.el6.x86_64.img
title Windows 7
rootnoverify (hd0,0)
chainloader +1
default ：指定默认的操作系统，如果为0，默认就是启动第一个title，如果为1就是启动第二个title
timeout：开机等待用户输入的超时时间，单位为秒，如果超时则进去默认指定的操作系统
splashimage：指定启动开机画面时的图片的位置
hiddenmenu：命令被使用时，不会显示grub菜单，会在超时时间到了后直接进入默认系统，按 Esc 可以看到 grub 菜单
title：给要启动的系统指定一个名字，这个名字会在 grub 菜单中显示
root：指定 linux 系统所在的引导分区
kernel：指定 Linux 内核的地址
initrd：指定初始内存盘的地址
rootnoverify：指定 Windows 操作系统所在的引导分区
chainloader：+1表示从指定分区的第一个扇区查找引导文件

三、 MBR、主分区、 扩展分区、 逻辑分区、物理分区、活动分区

1、MBR：全称为 Master Boot Record(主引导记录)，一般理解为 类似于 linux 下的 GRUB 的引导程序(其实引导程序只是MRB的一部分)，
     由三部分组成：主引导程序、分区表、主引导记录有效标志字，大小为 512 byte，主要作用是将  操作系统自己的引导程序 读入内存，并跳转到操作系统的
     引导程序

2、主分区：安装操作系统的分区，一个硬盘最多可以有四个主分区

3、扩展分区：主分区之外的分区

4、逻辑分区：在扩展分区上划分的区域，最少有一个逻辑分区

5、物理分区：逻辑分区是软件上的分区，物理分区是硬件上的分区，它可以将一个服务器划分为多个处理单元，每个处理单元相当于一个
     小型的服务器

6、活动分区：指的是这个分区可以做为系统引导分区，一般C盘会被标记为活动分区，U盘也属于活动分区
