ArcGIS Javascript API 1.2 Released

Apr 7, 2017

    ArcGIS Javascript API 版本更新到1.2，页面中增加：

<script type="text/javascript" src="http://serverapi.arcgisonline.com/jsapi/arcgis/?v=1.2"></script>


    就可以开始使用1.2最新功能，在线例子中增加了相应接口的使用方法，具体更新包括：

    1.支持 Image Services

    新增esri.layers.ArcGISImageServiceLayer接口可以调用栅格数据集或ArcGIS Server Image发布出来的影像服务，和矢量图层一起在客户端绘制显示。

    2.支持 Dojo 1.2

    Dojo每次更新都吸引无数眼球，除了EXT之外大家对这个商业支持力度最大的项目还是满怀希望，解决bug提升效率，毕竟它的license许可更受欢迎。

    3.动态显示缓存过的图层

    动态图层设置max scale和min scale是很自然的事情，目前直接对缓存过的图层提供了类似的功能，支持指定地图缩放级别，来显示缓存服务，过去需要在一个地图服务中进行设置，现在多个服务（动态图层和缓存图层）也能够相互切换了。

    4.手动刷新地图

    当对数据进行编辑之后，为了让客户端能够及时看到最新的实时数据，需要手动执行refresh方法刷新地图。

    5.改进Graphics图层性能

    地图上存在大量Graphics时效率会变得低下，现在对性能做了一些改进，究竟能达到什么程度有待测试。

    6.动态投影

    过去Javascript API实现的地图应用，其投影由基础图层所决定，现在可以动态投影来匹配空间参考。

    7.Geometry服务更新

    esri.tasks.GeometryServices的labelPoints方法允许在多边形中指定Label显示的位置点。

    Bug修复：

NIM034356 Double-click zoom is a default navigation type for the JavaScript API at version 1.2. NIM036958 Style Attributes (STYLE_DASHDOTDOT and STYLE_NULL) of SimpleLineSymbol now work as expected. NIM036833 Graphic.attribute names that contain decimal points (mytable.area) can be used in an InfoWindow through the Graphic.InfoTemplate. 

    注意：一些新功能需要ArcGIS Server 9.3 sp1支持，最好打上sp1补丁以配合js api 1.2使用。 本文转自Flyingis博客园博客，原文链接：http://www.cnblogs.com/flyingis/archive/2008/11/26/1341461.html，如需转载请自行联系原作者
