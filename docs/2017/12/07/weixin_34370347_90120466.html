更快更低耗！Spread表格控件V11 CTP发布，性能巨大提升

Dec 7, 2017

作为一款功能和 Excel 类似的表格控件，Spread Studio 提供灵活的定制能力和丰富的数据可视化效果，可用于在应用系统中实现表格数据录入和编辑等交互功能，涵盖 Windows Forms、ASP.NET、XAML 以及 WinRT 各平台。
 
Spread Studio 素来以数据处理快、内存占用低、和Excel高度相似而享誉全球，被中国石油、华为、NEC、宝钢、中国农行等用户广泛应用在各领域的信息系统中。
 
日前，全新的Spread Studio V11CTP版本强势发布，为您呈现更惊艳的性能表现！本次更新在Spread for Winforms中引入Spread Common，这是Spread .NET平台的公共组件，帮助Spread产品从底层提升性能，带来巨大提升。
 
下面，我们将通过 Spread 处理3个各有特点的文件，带您切身感受一下新版本在性能和内存方面的巨大提升。

 

测试环境

操作系统 Windows 10 企业版
内存 4G
CPU i7-3615QM 2.30GHz 2.30GHz

 

1、处理大数据量的文本

测试用例：用 Spread 处理有3个sheet、65000行和10列的文本文件，新版本导入仅需1秒，导出仅需2秒，内存占用仅5MB！

 

2、处理含丰富样式的文件

测试用例：用 Spread 处理有100个sheet， 1000行和20列含有背景色边框的单元格。新版本导入和导出分别耗时仅为7秒、10秒，内存占用仅50MB！

 

3、处理含复杂公式的文件

测试用例：用 Spread 处理有500行和80列的文件，含复杂公式的单元格，例如VLOOKUP函数。新版本导入和导入仅需0.5秒和0.4秒！内存占用不到5MB！

从测试的结果中，可以直观的看到：

Spread 在进行大数据量的导入导出时，时间消耗大量减少，同时内存消耗急剧下降，这也表明 Spread Studio V11 CTP 版本的性能得到了巨大的提升。
