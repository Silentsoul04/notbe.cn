Rabbitmq~linux环境的部署

Dec 7, 2017

之前写过在windows环境上部署rabbitmq,这回介绍在centos上对这个消息中间件进行部署的过程

一 下载和解压

wget  http://www.rabbitmq.com/releases/rabbitmq-server/current/rabbitmq-server_3.6.10-1.debian.tar.gz

tar -xzvf rabbitmq-server_3.6.10-1.debian.tar.gz

二 安装依赖包

yum -y install xmlto

http://www.erlang.org/downloads

找到最新的erlang安装包,otp_src_20.0.tar.gz,可以下载后,直接解压它

 tar -zxvf otp_src_20.0.tar.gz

三 配置环境

进行otp_src_20.0目录,执行以下命令

./configure --prefix=/usr/local/bin/erlang --without-javac

如果执行没有成功,说明我们没有依赖组件,需要重新安装一下组件

yum install -y gcc-c++
yum -y install ncurses-devel
yum install openssl-devel
yum install unixODBC-devel
yum -y install gtk2-devel binutils-devel

四 再次运行配置与安装

./configure --prefix=/home/erlang --without-javac

make && make install

五 配置共享路径

export PATH=$PATH:/usr/local/bin/erlang/bin:/usr/local/bin/rabbitmq_server-3.6.1/sbin
重启并生效
source /etc/profile
 
下面我们就可以通过客户端去连接它了,开放端口是5672.
