树莓派的.bashrc和.bash_aliases文件

Oct 18, 2017

在你的home文件夹中，你能够找到一个包括用户配置的隐藏文件.bashrc。

你能够依据自己的须要改动这个文件。


文件里为你提供了一些实用的调整设置。默认情况下当中一些设置是被凝视掉的。


比如，一些ls命令的别名：

alias ls='ls --color=auto'
#alias dir='dir --color=auto'
#alias vdir='vdir --color=auto'

alias grep='grep --color=auto'
alias fgrep='fgrep --color=auto'
alias egrep='egrep --color=auto'


类似上面提供的别名是用来帮助其他系统的用户熟悉使用（比如dir相当于DOS/Windows系统中的ls）。另外一些是为了给像ls和grep这些命令的输出结果加入颜色。


当中也提供了很多其它ls命令的变体别名：

# some more ls aliases
#alias ll='ls -l'
#alias la='ls -A'
#alias l='ls -CF'


Ubuntu用户应该对这些设置更加熟悉。由于这是Ubuntu发行版本号的默认设置。取消这些命令的凝视就能够让这些别名在稍后生效。


以“#”开头的命令行表示被凝视。为了启用它们。仅仅须要将“#”移除，等下次启动你的树莓派就会激活这些设置。


这里相同有关于.bash_aliases文件的引用。默认情况下这个文件并不存在：

if [ -f ~/.bash_aliases ]; then
    . ~/.bash_aliases
fi


这里的if条件用于在导入这个文件之间检查文件是否存在。


你能够创建这个.bash_aliases文件，并在里面加入很多其它的别名。比如：

alias gs='git status'


你也能够直接加入其他东西到这个文件里或者是其他文件。仅仅要将文件像.bash_aliases文件一样包括进来就能够。
