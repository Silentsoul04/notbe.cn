linux服务之NFS和SAMBA服务

Oct 19, 2017

         这几种网络文件传输最适合局域网。网络中用FTP

           一：NFS服务

            nfs(network file system)网络文件系统，改服务依赖于rpcbind服务。client通过rpc訪问server端的rpc。nfs通过rpc来申请port号。rpc将分配的port号映射到portmap表中。然后将port号发送给client的rpc.

               安装的包有：nfs-utils  rpc包。

               port号： rpc的port为 111    nfs的port为 2049

               配置步骤：

                                  1：/etc/exports  这个文件控制訪问的用户和读写权限。

                                  2：启动rpcbind服务。service rpcbind start

                                  3：启动nfs服务，service nfs start

                                  4:  client查看server开放文件。showmount -e 172.16.0.25

                                  5:  rpcinfo 查看依赖服务的开放的port信息。

                                  6：client挂载nfs，mount 172.16.23.253:/share /mnt


     二：autofs 自己主动挂载文件系统

             autofs自己主动挂载文件系统能够非常方便的挂载文件系统，能够设置超时时间。

                   配置步骤：

                               1：安装服务包autofs。

                               2：配置文件/etc/auto.master,依照配置文件里的语法来设置自己主动挂载文件夹。样例 misc文件夹

                               3：将挂载配置文件auto.misc复制一份，来对自己主动挂载文件夹进行设置。

                               4：路径以/开头的必须在前面加 :  。

如挂载windows的c盘  ://172.16.23.2:c\$ 。\为转义符号。

                               5：查看挂载文件夹自己主动出现。



      三：samba服务：

                  samba服务用来作为Linux与Windows系统之间的文件共享。最早是linux模仿windows的协议。一般用linux的client来连接windows的共享文件。

                   samba的port号：139 和 445

                    配置步骤：

                                  1：安装samba服务  samba-server  和客户端 samba-client软件包

                                  2：配置文件设置Linux共享文件/etc/samba/smb.conf

                                  3:  samba有匿名登录方式，和用户登录模式。Linux作为client连接windows的共享文件方式：

                                               a.   smbclient -L 172.16.24.235  扫描windows上共享出来的文件。



                                               b.   smbclinet  -U administrator //172.154.24.12/share /mnt 挂载文件系统，

                                               c.  mount  -o username=administrator,password=123 //172.16.12.2/share /mnt
