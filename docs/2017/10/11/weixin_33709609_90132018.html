QTP的那些事--QTP无法执行QC上的用例（转）

Oct 11, 2017

问题描述：

现象一，无论用户是登陆到QC执行QTP自动化用例，还是通过QTP打开QC上面的business/scripted component用例直接执行，系统都会抛出如下Error。

Run error Invalid procedure call or argumentLine (1): "---".

现象二：QTP已经打开了一个Component，试图打开第二个component的时候QTP提示：Your Quality Center session has been disconnected. Contact your system administrator for more information.

环境描述：

已经安装好了QC9.2,QTP9.5, 并且已经安装了QuickTest Add-in for Quality Center这个插件。具体环境准备与安装请参考QTP installation guidance.doc，TAO Install Configuration(Setting the ShareFolder).ppt等。

分析过程：

1） 针对现象一，通过查阅相关文档，得知这是QTP9.5 release 自身存在的一个问题，需要进入到QTP安装所在目录（e.g. C:\Program Files\HP\QuickTest Professional\Bin），使用新的TDSerializeMgr.dll(dated 3/20/2008)替换掉同名的dll文件。.

2）上述问题也可能是由用户计算机的语言设置前后不一致导致的。

具体解决方法如下：在Regional and Language options 中，把"Regional Options" tab 和“Advanced”tab 都设置成“English(United States)”。
