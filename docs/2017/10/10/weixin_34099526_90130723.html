iPhone不再孤独，Palm Pre为伴——互联网的未来在手中。

Oct 10, 2017

整整两年，两年前的今日，2007年1月9日，伟大的iPhone诞生了。就是今天，当我温习那次革命般的发布会，仍然很激动，仍然被乔布斯的激情所感染，甚至有点热泪盈眶。特别是看着乔布斯在揭开iPhone前，极为克制、压制内心激动的姿态与神情，再回想这两年来iPhone便在五花八门的手机市场如一骑绝尘入无人之境，我想两年前乔布斯一定知道当他把“iPhone”打在屏幕上时意味着什么：未来。还记得当时他说重定义了手机，革了手机的命。那时那刻有多少人相信？此时此刻又有多少人不信？

那次发布会已成经典，是我见过乔布斯最精彩的演出，巅峰之作，不仅仅是产品本身的伟大。也就是从那一天开始，乔布斯征服了我，我成了苹果的粉丝。

从2007年1月iPhone面世，到2007年iPhone上市，再到2008年iPhone的App Store年度创新之首，两年了。对于日行千里的IT，两年是一个很长的时间。在这两年内，我再也提不起对任何手机的兴趣，无论对硬件还是对软件，包括曾经满怀希望的Gphone，很快也发现实在是个挂羊头卖狗肉的东西，根本不可能和iPhone相提并论。我一直很纳闷也很郁闷，一骑绝尘的iPhone映射苹果伟大的同时也映射着这个领域其他人的苍白。我并不希望看见这样的局面，这样会产生惰性，你看，iPhone 3G算什么？还有正浮出水面的iPhone Nano，又算什么？除了App Store值得一提，iPhone在硬件和自身软件上已经停滞了，已经开始吃老本了。苹果需要对手，iPhone需要对手，我希望看见对手，竞争与较量才能加速成长和发展。难道乔布斯两年前“领先五年”的狂妄是事实？iPhone画出了美丽的未来，互联网的未来，但是仅仅iPhone一款产品，仅仅靠苹果一家企业，是无法独立承担起一个行业的未来。我还要等多久？三年吗？明明未来就在眼前，却抓不住，你说有多着急！

就在我已经调整好心平气和的心态之时，今天，Palm Pre发布了，Palm不具备Apple的号召力，Palm Pre更不比iPhone响亮。整个业界的反应也很低调，淡淡的描述着此事。如果不是今天偶然登录twitter，偶然看见淘宝的正淳同学正论及此事，也许我就错过了。他说：“http://tinyurl.com/9cmjrn看完之后，pre让我对iphone完全没兴趣了。”据称淘宝的UED团队半数以上都拥有iPhone，能发出这样的感慨，或许真有点不一样。我没抱太大的希望点了进去，去怀着满满的希望从视频中走了出来。

怎么说呢，很激动。其实从发布会本身来说没什么可圈可点的地方，就是一个人在一张桌子面前演示，背后一个大屏幕显示演示画面，不能算演出只能算演示。Palm也许知道，它没有像乔布斯这样充满个人魅力的人来释放人的激情，来秀一场精彩的演讲，于是它明智地扬长避短，用最大的演示画面，让所有人的目光专注地聚焦在产品本身上。朴实的演示却掌声不断，这些掌声都是送给产品的。

Palm Pre的硬件UI个人认为很普通，甚至一点都不时尚也不养眼。既然选择了很酷的纯黑钢琴镜面，却搞成活泼的圆头圆脑（连屏幕也是圆角……），再配上五彩缤纷的系统UI。视觉上感知很乱，既缺乏iPhone的色彩反差，在色彩和形状的视觉认知上也不统一，用流行的话说：山塞感非常强。在这点上说实话iPhone的整体设计真的是天衣无缝：有一点圆的金属边框围着黑色，硬朗的直角屏幕，背面银色金属配黑色，正反都有合理合适的色彩反差，系统UI也是华丽而不失稳重……（iPhone 3G不算，背面太糟，同样的错误）。说到这里我小跑一下题，我对iPhone 3G的背面真的很失望，本来我以为iPhone二代会更帅更强，所以不是很爱惜一代，结果3G倒饭的背面和无变化的硬件配置让我有点悔没好好爱惜竟成为经典的一代，然后破罐破摔了，唉……

滑出的实体键盘是另外一个问题，我并不认为iPhone的虚拟键盘多么不方便，甚至很习惯，可能是我没用过实体键盘的原因。不过我想，在这么小的键盘上单手或双手输入，都不可能也不可能比上十指输入（哪怕是侧滑的Gphone），在手机上强调输入一定是件痛苦的事情，手持设备的文本输入注定只是辅助。既然这样，实体键盘比虚拟键盘方便又能方便多少呢？为了这不常使用的应用的力反馈手感？损失是无法得到更大的屏幕和更薄更简单的机身，而显然后者比那一点点的手感和方便性重要得多。所以我非常认同iPhone的虚拟键盘设计，利远大于弊。手持设备的键盘输入永远不会是重点。甚至我认为哪怕电脑，虚拟键盘代替实体键盘是未来的方向，我一直很期待放大的，能够双手输入的iPhone。IBM的小红点宣告了鼠标的死亡，而多点触摸的出现我认为实际上已经昭告了输入的未来方向。所以Palm Pre安置实体键盘我认为不好，当然这也和两家公司的历史有关，Palm有众多的传统用户，他需要承上启下的照顾用户使用习惯，渐进的转变，而Apple则没有历史包袱，可以培养用户全新的习惯，很明显，我已经接受了，哈哈哈哈。

Palm Pre的硬件设计批完了，实际上我也是很麻木的看完它的硬件UI，谈不上失望因为没有过希望。麻木的看完硬件，却被其系统及软件震惊了，然后遗憾它的硬件设计，配不上软件设计。

这是第一次看见一款手机能够带来和iPhone对等的触摸操作体验，同样的流畅和润滑，甚至有众多iPhone不具备的创新体验。可谓出于蓝甚于蓝！iPhone最杀人的就是它的操作体验，虽然我们知道iPhone不完美，每个人也都能想象更完美的iPhone，但是只能想象。而Palm Pre最成功的是，它是第一个宣布iPhone操作体验不完美的成品手机。看看iPhone的不完美：

波浪形的Launcher
iPhone 1.0时代的人应该记得Dock这个炫目的软件，但是非iPhone原生应用，除了炫机其易用性也有问题，而Palm Pre原生设计了，从视频上看应该既炫也易用。
卡片进程
OMG，这个实在太帅了，这是iPhone应用的巨大缺陷之一！像iPhone Safari 多窗口那样浏览，太赞叹了，而且还更强能够管理，Safari连多网页都无法管理，它居然能管理多应用，而且非常流畅。能移动切换应用排序，帅！关闭是抛出屏幕，帅！即便在窗口情况下也是动态运行状态并可控，帅完了！

本来我还想说一些，比如轨迹球比iPhone的按键要帅也能想象出来其丰富的应用点和易用性。但是我觉得最靓丽的，最创意的就是卡片进程，在这点上抛开了iPhone，感觉有本质的区别。iPhone基本上属于单线程，应用一个程序前需要关闭另一个程序，即便现在有些多线程的三方软件也是噱头更多。目前iPhone的系统和软件有非常明确的界限，至少在感知上是如此。而Palm Pre顺滑的卡片进程下，模糊了系统和软件的界限，至少在感知上是这样。随时调出应用或切换应用，而不是启动应用，随时返回桌面而不是关闭应用，多么帅，是不是更接近电脑。

从视频上看，iPhone所拥有的杀人UE，它全部都有而且更炫，而卡片进程和实体键盘的存在使得它还具备更多iPhone不具备的UE，单从iPhone最引以为豪的UE上，可以说已被完美的超越。我非常开心的看见iPhone终于迎来了强大的对手，Palm这一拳打得着着实实的不错！

突然想到，Palm Pre如出一辙的同样选择了1月9日这一天发布，这是巧合吗？

UE被超越不等于被全面超越，iPhone最大的优势是它已经两岁了，而Palm Pre才刚现身。两年的时间，Apple围绕iPhone已经设计并形成了一个全面的整体战略，iPhone不是一个人在战斗，陪伴的还有iPod Touch，MobleMe，App Store，iTune，iWork，Mac，配件等，无论成熟不成熟，它都在成型中。我是一个看重整体性的人，局部的丢失可以弥补，而整体的丢失无法挽回。目前有限的信息看来，Palm Pre对iPhone的超越很有限，只是在手机这一点上，并且就手机这一点我个人认为也只是在系统UE上超越，硬件UI完全比不上，系统及软件UI也略有逊色，只是接近。Palm Pre的路还长，但它出现了，算是给了骄傲的苹果当头一棒：“你小子继续拿iPhone 3G和iPhone Nano忽悠嘛……”

这不，同样是今天传出来的消息：“在Macworld会场中，广泛流传着这样一则消息，苹果下一代iPhone手机将搭载4核心处理器，提供强大的多媒体性能。”我还是选择相信和等待下一次的“iPhone 4C”，我相信iPhone的UE比Palm Pre差不是能力问题，而是两年前的硬件性能限制问题。

我想每个拥有iPhone的人都不会认为它只是手机，甚至智能手机，我也不知道如何定义它。Google加入了，大家都知道与其说带来了Gphone不如说带来了Android，尽管我认为Android目前看起来很烂，但是至少比所有的传统手机系统先进，适应时代的发展，未来众多山塞手机配备后也能实现并支撑丰富的现代手持应用，这能提高高质手持设备的普及率。现在作为传统三大主流掌上系统的Palm也宣布加入，Palm Pre也许不是那么完善，但是其系统Nova绝对顶级，看起来比Mac OS美，更不要说看起来垃圾的Android了。现代掌上系统已形成Mac OS、Android、Palm三足鼎立，剩下的两家传统主流Symbian、WM想来不会远了。

哎呀，说到WM了，微软……顺便抨击一下。几个月前，微软把IE6搞进WM，我简直吐得一塌糊涂……还嫌IE6祸害面不够广不够久……唉，微软老了，这么多年来总是慢N拍。我要说微软坏话了，在博客园可能不合适，但我还是想说，想当年，2004年微软的C# 1.0，提出的智能客户端概念让我无限憧憬并展开学习，那时我想像着06年Vista发布之后，所有软件都融入互联网，结果空头支票。随后微软的Live概念又让我一震，结果还是难有起色不了了之。在互联网及周边，微软东一枪西一炮，枪枪落空炮炮哑炮，唉。我是一个热衷于当小白鼠的人，我曾经说我只用alpha版软件，我希望从中捕捉和感知未来。现在Win7来了，我完全没了兴趣，我从微软身上看不见未来了，这个在2008年9月份把2001年的IE6加入WM的家伙……

Englive.cn的意见反馈中，不少用户询问为什么不支持其他手机？我无奈的回答：“目前支持iPhone和Gphone。普通手机的互联网处理能力太差，基本只能浏览图文，很难实现交互应用，所以暂不支持。”Palm Nova来了，很惊喜的知道其浏览器同样基于webkit核心，真TMD帅！难道webkit要一统江湖了？从目前来说这可是个好消息！还有用户希望我们提供供安装的软件，我不知道怎么说，或许现在怎么说普通用户都无法接受，因为软件安装是个很深厚的习惯，哎呀这个话题展开又会很长，有兴趣的可以看看两年前我写的一篇博：《从2005年2月的一封信谈开，闲聊互联网。》，简单的说，软件必将融入互联网，所以我从业八年来坚决不碰独立软件，我认为那是没有前途的，即便离开浏览器做客户端，那也要是基于互联网的离线客户端。今年这种思路有了个忽悠的专业名词：云。

现在3G、WIFI的普及力度在加大，手持设备的硬件及软件也逐渐跟上了，大环境越来越成熟，这意味着iPhone两年前宣告的未来终于就在眼前了，就在前方了，近到可触摸，是什么？互联网真正从桌面向手中转移，从定点向不定点转移，从有线到无线再向无处不在转移。这不是长期以来阉割的移动互联网，而是真正的完整的互联网，降临在手上。

展望前方，无限美好。这就是互联网最诱惑的地方：永远不缺精神粮食，永远不缺美丽未来。

此外，Palm Pre 第三个重要原因是其运行的基于 WebKit-based 的软件平台 Web OS ，它之上运行的客户端是使用 HTML 、CSS 和 JavaScript 语言编写的。Palm 表示将发布 SDK 并推出应用商店，可以肯定以后会有许多第三方应用出现。

OMG，整个系统的表现层就是一个WebKit浏览器，这简直太帅了，还不知道是否支持Flash，真正是血统比较纯正的WebOS。

四年前选择Web标准并如一日的坚持和挖掘，值！给我一个现代浏览器，我能重设计地球。
