powerdesigner连接oracle数据库，PDMReader导出数据库字典（转）

Sep 4, 2017

二期项目的开发工作已经全部结束，这边客户对于项目验收的流程又有自己的一套规则。原先的办法不能套用，只能因地制宜的配合和推动、完成项目的验收工作了。

技术测试：

技术测试需要包括六个方面，会议时间长达三个小时。。。

在此的第三项系统数据库的设计文档。记得客户总公司【IT】信息技术部的同事说需要我们提供数据库的【PM】图。挂了电话，我才意识到【PM】图，我不清楚啊。但在给客户打电话过去，似乎显得很不专业。问过同事，他们却也不知道，什么是【PM】图啊！

能想到就是【PowerDesigner】，但即使缩写也只能是【PD】图啊。想来想去，时间紧迫还是打电话再给客户【IT】信息部的同事确认下，问清楚他们需要什么吧。也怪了，打过去电话，没有人接。。。于是想想数据库验收相关的。数据库设计文档、数据库字典、PowerDesigner、PDM。。。

于是找资料，装软件，问同事寻求帮助，把能想到的都做出来。累啊。

【解决】

一、安装【PowerDesigner】

自从离开鸟窝，就没有再用过【PowerDesigner】了。现在设计数据库有专门的同事负责，再温习下吧。

版本【powerdesigner V12.5】。下载汉化包直接覆盖源文件重启就好了。

【PowerDesigner】不能使用直连的方式连接【ORACLE】数据库，只能使用数据源【ODBC】的方式连接。

那么接下来要做的就是配置数据源。

菜单--》文件--》逆向工程--》Database...

在New Physical Data Model 窗口中，DBMS(Database Management System)选择 ORACLE 的版本

在Database Recerse Engineering Options窗口中选择连接数据库的类型，这里选择 Using a data source

点击Connect to a datasource 按钮

选择ODBC machine data source 机器数据源

之后配置数据源

点击下一步，数据源类型为默认的文件数据源

选择驱动程序，oracle

下一步


完成数据源的配置

测试连接

测试OK，返回到主界面，选择数据源，输入用户名密码点击 Connect

之后选择表就可以了。如果表较多的话，时间会长点

PowerDesinger的报表导出功能太强大专业了。

但和现在客户要求的不一样，所以不做过多累述。

【二、安装PDMReader】

PDMReader能够读取【PowerDesinger】生成的【*.pdm】文件

在此可以导出数据库字典，html格式和doc格式。

技术部的验收工作已经完了。会议上到讲解数据库了，我问客户你要数据库字典还是还是数据库关系图...

他说给我【PDM】图就好了，我只能心里说句，哥哥你真专业！
