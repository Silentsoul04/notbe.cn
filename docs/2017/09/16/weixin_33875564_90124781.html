linux 如何才能在个人使用领域战胜 windows

Sep 16, 2017


    思考linux如何在个人使用领域才能战胜windows,这个念头是先前有过,但没有深思.最近一段时间,因为ubuntu 11.10发布,fedora 16发布,再加上我断断续续地在整理先前Solaris使用中记录的一些笔记.因此慢慢开始考虑这个问题. 
    linux 在个人使用领域战胜windows.这是个真命题还是假命题呢?很难说.至少从目前来看,短期内(5年内)可以把这个看成一个假命题. 
    印象中刚刚接触linux时侯,从网上看到有不少技术人员嘲讽windows的一些缺陷,然后大吹特吹地说linux有多好,仿佛linux就是操作系统界的曙光和明天.接触linux这么久,从一个普通使用者的角度来看,linux要取代windows,成为操作系统的霸主,还有很长的路要走. 
    每个操作系统都有它的适用范围或者是适用群体.windows是为了让普通人能够使用,面向大众,特别是非技术人员.linux最初是面向技术人员,特别是和编程相关的人.因此,从源头上来看,linux和windows走的就不是同一条路. 
    现在的linux发型团体或者厂商,现在在大力推广linux,推广给谁,大部分还是普通的使用者,不是什么技术牛人.因此,linux要战胜windows的关键,还是要满足大多数普通使用者的胃口. 
    再说大多数人对操作系统的使用认知.大多数人对于使用什么操作系统不是特别关注的.他关注的是操作系统上大部分常用特性的易用性和软件资源的丰富程度. 
    举两个例子: 
    1. dsl上网,这个在windows里面,只要去桌面右下角通过点击网络设置,就能一步步地配置好dsl上网,但是在大多数linux系统中,dsl上网的配置在桌面系统中很难找到,甚至专门要去启用pppoe-conf. 
    2. 在windows中安装一个播放器,安装好就能播放视频或者音频文件,一般不需要再去下载解码器.但在linux中,安装好播放器之后,还要去找ffstream等一系列解码器. 
    其实通过这两个例子,可以看出,linux在使用上的友好性上,比windows要差.软件资源上就更是如此了.好多windows上的软件,在linux上都没有替代品,或者即使有替代品,可替代品的功能也是很差劲. 
    因此,不管是从使用的友好性或者从软件资源上来讲,linux根本就没和windows在一个档次上. 
    既然没在一个档次上,那么只能退而求其次了.想想办法让懂一些技术的使用者同时使用windows和linux.这个是现在一个比较现实的摆在linux厂商和组织面前的路. 

    所以,难怪ubuntu,fedora,opensuse在系统中自带了可以自动列出fat32和ntfs分区,并且在桌面系统中可以显示,点击后能挂载,打开后能修改其中文件的特性.
