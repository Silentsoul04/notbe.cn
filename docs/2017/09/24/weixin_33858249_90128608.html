Xamarin Alpha预览

Sep 24, 2017

好像是几周前，Visual Studio 2017发布了，我们已经在为下一个Xamarin更新。我们刚刚发布了下一版本的alpha预览版，我想和你们分享一些我最感兴趣的特点。如果你想帮助测试这些更新，请  切换你的IDE到 Alpha 通道， 并安装所提供的更新。

工作簿

我们最近精选练习册在这个博客，我们很高兴地展示最新和最伟大的工作簿已提供。包括在工作簿的α预览是C# 7包含Roslyn 2.0。对于修复，我们改进了与Android和iOS nugets支持Windows，伴随着各种性能改进。更多信息可以在我们的发行说明中在这里找到。

Xamarin Studio

我们很高兴能提供与 fastlane toolkit 来改善IOS的签署经验。快车道提供了自动化的证书管理，一个完整的解决方案配置，并签署了iOS应用程序，我们认为这将大大简化构建、测试和部署过程。而整合快车道仍在发展，这个最新版本签署它提供支持。

Xamarin.Mac

我们已经介绍了在 AOT编译的实验特征Xamarin.MAC，以及对TLS提供更新（TLS 1.2支持通过appletls）。你也会找到改进的NSNumber NSValue，绑定，和NSString。从 这里可以得到详细信息。

Xamarin.iOS

我们介绍了Xamarin.iOS内存使用和可执行大小的改进。在测试一个示例应用程序，可执行文件在减少~ 200kb，而被~ 400kb减少内存使用。 此外，我们增加了规则的Xamarin分析特征，防止IPA创作在调试模式下，和现在的项目包括天然WatchOS优化控制实验。 抓住所有这些和更多的信息来自 Xamarin.iOS release notes

Xamarin.Android

我们做了一些改进的快速部署，Xamarin。Android，现在我们正确处理超时和cancellationtokens AndroidClientHandler。支持TLS 1.2也可作为一种实验特征通过BoringSSL WebRequest。查看release notes（  这里 ）获得更多细节。

Odds and Ends

这是在一系列的Xamarin产品等多方面进行了改进，包括 Xamarin Studio 和 Xamarin for Visual Studio。反馈总是受欢迎的， and any bugs found can be filed on our 发现的任何错误都可以归档在我们的Bugzilla page。快乐编码！

注：此版本中当前不支持Visual Studio 2017。我们正在努力解决这个问题。
