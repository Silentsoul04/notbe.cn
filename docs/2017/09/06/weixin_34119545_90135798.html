QTP的那些事--The QTP computer is locked locked or logged off

Sep 6, 2017

原始地址：http://www.sqaforums.com/showflat.php?Number=457010

If the screen is unlocked (i.e. I can see the Windows Desktop) then everything runs fine.
If the screen is locked (it has timed out and gone into the screensaver) then all of my tests fail and I see the following message in QTP:
"The Quick Test Professional computer is locked or logged off".
Does anybody have a fix for this? My group would like to have the ability to kickoff scripts on this box remotely and at night, but it is constantly locking us out.

 

 

解决方案：

QTP will not executed on a locked system. From the FAQs at the top of the first page on this forum, a FAQ specifically addresses your question:
30 Problems with Running QTP on a "locked" system/keyboard
30.A http://www.sqaforums.com/showflat.php?Cat=0&Number=420045
The bottom line is that you can't let a QTP execution PC go into system locked mode. In the above thread there is a reference to a lengthy KB Mercury article (God only knows if this is still available on the current HP support site's KB), that blames Microsoft for the problem. Maybe they are right, maybe not.
There are various suggestions for software that will inhibit a PC from entering system locking, but in many corporations this will violate IT security policies, so use these with care and approval (if you are one who "asks for permission", rather than one who "begs for forgiveness" when caught).
And in the future, please: FIRST check the FAQs, then SEARCH second and lastly post what you think is new question, before posting again.
