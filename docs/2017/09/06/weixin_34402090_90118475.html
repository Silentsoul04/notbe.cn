【爆牙齿】微软的坟墓：Windows 7。（五）

Sep 6, 2017

Chrome这么好的产品在全球市场份额上进展缓慢。我认为这印证了两个问题：1、Chrome的设计理念太超前了用户无法接受。2、Google现阶段的重心根本就不是用Chrome来占领市场。我猜测，Google非常清楚，真正属于Chrome的时代还没到来，他清楚无论怎样宣传和推广都不能改变当前用户使用互联网的习惯，更不要说整个大环境由于IE的长年限制，从网站自身的表现形式、开发方式、运营模式都还没完全切入次世代。所以他基本放弃了对Chrome桌面市场份额的关注，全部精力投向了Chrome自身产品体系的完善。

Google在蓄势，在等待，等待时代来引爆和成就Chrome！

浏览器市场总结与展望

IE、FF、Chrome，目前最重量级的三大浏览器品牌已经基本说完，还有些枝叶需要丰满。

IE

全球市场：来自Net Applications的统计，全球IE6为23，IE7为18%，IE8为20%。我想如果其23%全球份额中去掉中国用户的采样，IE6的国外市场份额估计只是1x%。就算23%好了，当一个产品从90%滑落到23%，而又当一个产品以这样的方式从第一落到第二的时候，这是一个崩溃的临界点。以当今浏览器的快速发展，我相信对于国外市场，1年后IE6就会掉入15%，2年后掉入5%，基本消失。IE7是明显的过渡产品即没占领很大的市场份额也不可能再增长了，它大概也会在2年内消失。IE8和*IE9会强势增长。

国内市场：国内的浏览器市场有两个截然不同的统计。一是来自和上面一样的Net Applications对遨游22%的统计，二是来自CNZZ的详细统计。我不知道哪个准确，但我必须说我极度质疑CNZZ。在CNZZ的统计中，独立IE6的份额居然是63%，从02年MYIE发展过来的遨游居然只有4%，狗屁360居然5%，FF居然1%。换个计算方式，遨游日覆盖用户0.25亿，中国有效网民3.5亿，0.25/3.5=5%，但可能这么计算么？每天3.5亿全在访问互联网？所以CNZZ假完了，我觉得Net Applications对遨游22%的统计还相对靠谱一些。我个人感觉，在国内，独立使用IE（678）的份额应该是50%左右，而外壳的总份额应该占35～40%。也就是说在国内，IE总份额85～90%，其中一半是通过外壳使用。这90%的IE用户，我个人感觉，IE6:IE7:IE8=60%:20%:10%。由于国内IE市场可以说由外壳撑起了半边天，所以其趋势得后面从外壳着手分析。

Firefox

全球市场：来自Net Applications的统计，全球FF已超过25%。由于FF3.0和3.5包括后续3.6的产品差别远不如IE6和IE7的差别，再加上FF自动升级的存在和短生命周期，所以FF的不同版本基本可以视为同一产品。还是那句话，当一个产品以这样的方式拿到全球第一，也是一个临界点，和IE6截然相反，这是爆发的临界点。我认为在未来1～2年内，FF会保持快速增长，今年破30%应该没问题，然后逐渐放缓，长远来看，峰值会在40%左右。

国内市场：去年底，FF CEO接受腾讯专访时表示FF的中国份额5%，我个人认为略高一点点。FF在中国的普及难度很大，互联网行业外企的失败案例太多了，天朝专灭外企无论多优秀。我认为Firefox能在未来1～2年内达到中国份额的10%就很不错了。

Chrome

全球市场：来自Net Applications的统计，全球Chrome份额5%。我认为从今年开始，Chrome将真正进入增长高峰，前途无量，无法估计。

国内市场：1%？估计没有。不过Chrome的中国市场份额超过Firefox应该是没问题的，但肯定还是属于小众产品。

Opera

Opera 10.50真是很不错，相当不错！但是终究Opera是真正而彻底的小众市场，也完全没有进攻大众市场的资源和能力。它的成功和让人佩服的，就在于活着。

中国的外壳浏览器

我不知道国外的外壳情况如何，但感觉不会像中国的外壳这么辉煌和灿烂，而且用户接受度奇高。所以解放和拯救中国浏览器市场这个重担，Firefox和Chrome都扛不起来，只能寄望中国的外壳们，你们千万要争气啊！

Maxthon：遨游3.0采用了Webkit核心，作为中国最大市场份额的外壳，带头吹响了独立的号角。我很高兴，从我知道后，我就在英来网推荐浏览器Firefox和Chrome后面郑重的加上了遨游3.0，向英来网用户推荐。同时我也很郁闷，遨游3.0一直缓慢的开发，悠哉悠哉的开发。据说从2008年3月就开始了，那么都快两年了，仍然停留在alpha内测期间，非常低调，而且一直没有我所期待的跟进者。当然，我理解遨游的慎重，这螃蟹可不是轻松就能吃的。但是，丫也太慢了！但是，我还是只能点支持，好歹，中国有了希望出现真正的自主浏览器。但是，丫也太慢了！

Sogou：就在我对本文进行校稿修正的时候，搜狗浏览器成为第二个支持Webkit的外壳，这真是一个好消息！我哈哈大笑，这不是在配合我写文章么，真及时，呵呵，于是赶快把这段插入进来。我纳闷，遨游建立全国最大浏览器研发中心用了近两年时间进行基于Webkit核心的研发，在遨游3.0的开发日志中可以看到围绕Webkit核心所做的大量工作，而搜狗就这么悄无声息的丢出一个炸弹来了？这到底是遨游忽悠了大家，还是搜狗在忽悠？于是我开始检查搜狗，然后我觉得：搜狗偷奸耍滑投机取巧，没有老老实实开发，而是又做了一次Chrome 3.0.195.0的外壳。我估计是搜狗分析了Chrome的开源码，然后直接整合进了搜狗浏览器。相比之下，遨游是真正在啃硬骨头。

这该怎么说呢？我尊重遨游，但是也不能完全鄙视搜狗。Google对Chrome进行了开源，这本来就是可动用的资源，和Webkit一样，搜狗直接拿来主义也很正常。遨游3.0开发半年后Chrome才现身，我估计那时遨游非常两难，继续啃骨头还是变Webkit为Chrome？当然现在看来遨游选择了自主研发，而搜狗现在再披上一件外壳。我想遨游会在未来真正浏览器中占得一席之位，而搜狗不会。但是从“急功近利”的角度而言，搜狗浏览器会依靠Chrome去抢占市场，毕竟省了研发还拥有了最好的性能：在浏览器测试中，今天的搜狗浏览器2.0是遨游3.0的1.25倍，高出整整1/4。想想也正常，遨游只有Webkit而没有V8。

我赞成搜狗的做法，但我也为遨游加油！当然我也担心，遨游必须找准自己啃骨头的优势，并且通过搜狗预见到会有大量外壳直接快速转嫁Chrome，然后找出对付办法来。

但是，无论像遨游那样去啃Webkit的硬骨头，还是像搜狗那样直接快速转嫁Chrome，我相信，中国的外壳们要开始抛弃IE了。其中腾讯的套套我想应该是下一个跟进者。腾讯早就为QQ空间做了大量Gears支持，在Gears被Google抛弃后，将已实施的Gears技术过渡到HTML5是不错选择，而腾讯就需要为它的外壳浏览器增加HTML5来更好的支持它的QQ空间，这也可以成为一个技术营销点。那么除了开源的Webkit或开源的Chrome，还有什么更好的选择吗？

当遨游拥有了Webkit的速度，当搜狗拥有了Chrome的速度，世界之窗、360等还会坚持抱着IE的蜗牛大腿而丧失重要的营销制高点吗？中国IE外壳浏览器即将全面叛逃！

我之前说过，中国的IE市场由外壳撑起半边天，一旦外壳开始全面集体叛逃，IE如何立足？不信我们扳着指姆看IE在中国市场上兵败如山倒。说到这里，搜狗让我突然意识到自己刚才说错了一句话，那就是也许原生的Chrome在中国市场成不了大气，但是由于Google把Chrome开源了，所以，最终笑傲江湖，在中国消灭IE的也许仍然是Google！

Pivot：看见有人在评论里说到这个微软“最新”的浏览器。我懒得骂人，一笔带过一下免得某人继续在我每篇文章的评论中无知的喷：请先去学习一下什么是浏览器核心和了解一下各浏览器核心的优劣！Pivot发布的时候我就安装并使用过了，充其量就是一个官方华丽的IE浏览器外壳，它是使用低能的IE7或IE8核心，绣花枕头一包草。如果不是微软做的，兴许还有出路。

IE9

这才是微软将在三月份公开的新一代浏览器。那么IE9能救场吗？

首先从目前透露出来的零碎消息。第一、据说支持HTML5和CSS3，可是Acid3的测试才32分，比IE8象征性的增加一点点，有什么用？当然我得承认终于支持了圆角确实很好。那是不是每个版本增加一个css3的支持？第二、官方大力吹捧的硬件加速2D渲染，且不说绑定了DX，实际效果也尚待检验。第三、新的js引擎，这倒是比较利好的消息。嗯，具体功能待三月份再看，其实我觉得没多大期待，因为……

我们假定IE9完全做到了当前Chrome或Firefox的水准，当然不可能，我是说假定。那么IE9正式版什么时候发布？至少一年后吧，这都是乐观估计了。那么到2011年的时候，0%市场份额的IE9面对的是30%市场份额的Firefox。你认为IE9需要花多少时间吃掉IE678？能吃掉吗？我琢磨，当IE9达到10%市场份额的时候情况是这样的：
2012年底，如果我们还没灭亡，我预言浏览器的全球市场份额：
IE6：5%；
IE7：5%；
IE8：15%；
IE9：10%；
FF：35%；
Chrome：20%；
剩下的：10%；

你认为在可预见的，混乱的IE版本市场下，IE还能有还击之力吗？终将被逐个击破，且越往后越发的兵败如山倒。所以我之前说IE已无力回天，IE9也无法救场。微软在浏览器市场上的败局已成定型。

每次帮我预览文章的业余人士对我说：“你阐述了很多关于你对浏览器市场的观点过后，再回过头看你的大标题，很容易让人给出这样的评论：浏览器嘛，不过是MS的一点渣渣。”其实评论中早就出现类似观点了。在众多评论中，除了大量跑来露屁股的“人”（我统统保留，欢迎继续并鼓励相互攀比），也有个别反驳得我点头的论点，我想我还是先把自己的想法写完。关于我花了那么多笔墨去写浏览器，是因为我认为：小小的浏览器就是蝴蝶的小小翅膀。

噢，留意没？我忘了Webkit的正牌主人：Safari。
