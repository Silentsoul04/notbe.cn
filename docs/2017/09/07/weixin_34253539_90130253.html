测试管理工具QC第一篇-QC介绍

Sep 7, 2017

工欲善其事，必先利其器，说到项目的管理，我们测试人员首先想到的是一个最流行的工具Quality Center,但是最新的11.0的版本（应该是截止到现在的2011年吧还是11.0的版本），却让HP总是在更改工具的名称，叫了ALM(application lifecycle management).真是大气啊。。

以下是安装的预先环境介绍：（摘自陈能技老师的博客）

QC11 premier Edition 安装注意事项

安装环境：Windows Server 2003 + SQLServer 2005

系统环境需求可查看：
https://h10078.www1.hp.com/cda/hpms/display/main/hpms_content.jsp?zn=bto&cp=1-11-127-24^9679_4000_100__&jumpid=go/TDQC_SysReq

不支持IE6：
Microsoft Internet Explorer : 4.0 (compatible; MSIE 6.0; Windows NT 5.2; SV1; .NET CLR 1.1.4322; .NET CLR 2.0.50727) is not supported!
Only Internet Explorer 7 and Internet Explorer 8 are supported.
解决办法：
1、改用IE7、IE8
2、改用almexplorer：
http://update.external.hp.com/qualitycenter/qc110/others/almexplorer/index.html
但是需要安装：
.NET Framework 3.5 Service Pack 1
Microsoft Visual C++ 2005 SP1 ATL Security Update Redistributable（vcredist_x86.exe）

默认没有安装DemoProject，需要从光盘导入：
G:\ALM\DemoProject\ALM_Demo_Project.qcp

DemoProject的登录用户名改了，后缀从原来的qc改成alm了，例如alice_alm

关于QC11的新功能可观看附带的视频：
http://localhost:8080/qcbin/Help/welcome_movie/video.html
