让32位系统支持4G内存(仅限vista/xp)

Sep 7, 2017

相信好多朋友近期都会有加大内存的冲动了吧，毕竟内存便宜了不少，
但咱们都是惯用的32位系统(vista/xp)一般情况下只能认到3G内存，
那么，如何才能不用浪费多出来的内存呢(再怎么便宜也是花钱买的呀)
小弟上网查找了一些相关的文章，总算找到了一些缓解的方法。
就和大家分享一下吧。

理论上32位操作系统是完全支持4GB内存，当然4G也是32位系统支持极限了。
如果那位兄台是用4G以上的话，那还是建议你用64位系统算了 

咱们首先检查主板是否支持，下载everest检测软件  右击使用迅雷下载
在主板－内存一项中，Physical Address Extension（PAE）查看“操作系统支持”“处理器支持”是否为“是”，
如果是“是”的话，代表主板支持(如下图001)；但如果是“否”的话代表主板不支持，下面的就不用看下去了。


WINDOWS XP 用户：
右键点击"我的电脑"，顺次点先 “属性－高级－启动和故障恢复－设置－编辑”
然后在弹出的记事本中查找以下内容：
\WINDOWS="Microsoft Windows XP Professional" /noexecute=optin /fastdetect

在"icrosoft Windows XP Professional ”加上"/PAE” 即：
\WINDOWS="Microsoft Windows XP Professional" /PAE /noexecute=optin /fastdetect”

这样保存关闭。


WINDOWS VISTA 用户:
顺次点击 "开始"－"所有程序"－"附件"；右键点击"命令提示符"点选"以管理员身份运行"
在 命令行窗口 中输入下述命令回车即可：
BCDEdit /set PAE forceenable

重新启动后，电脑属性的下方多了“物理地址扩展”几个字，虽然显示的还是3.25g内存，但实际上4g内存已经完全启用了，但怎么证明这一点呢？就用ramdisk虚拟硬盘软件来测试。

ramdisk是一款将内存转换为虚拟硬盘的小软件，内存的速度是硬盘的几十倍，所以在我处理ps等需要大量暂存盘的软件时，通常都会用它来充当软件的暂存盘，这样软件的速度会倍增，在系统没有开启PAE只能识别3.25g内存的情况下，使用ramdisk会直接占用这3.25g的内存，而开启PAE后，用ramdisk设置768mb的内存作为虚拟硬盘，再调出任务管理器，会发现内存使用丝毫没有减少，也就是说，ramdisk调用的768mb内存是系统没有显示出来的768mb，言之，系统已经找到了4g内存。
