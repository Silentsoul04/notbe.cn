架设自己的WebDAV服务器作为AutoCAD WS的数据存储

Jun 26, 2017


AutoCAD WS作为一款免费的在线CAD软件，不仅可以在浏览器上运行，更可以在移动设备比如iPad/iPhone/iTouch和Android设备上运行，可以说是一个变革。AutoCAD WS完全运行在云平台上，对浏览器而言本地无需下载任何插件就可以使用。此外AutoCAD WS还提供了基于云的存储空间，更便于文档的共享和协作。相信不少人已经开始使用了。

 

但同时我也看到大家不少的担心，我们的数据放在云上安全不安全？我们的数据基本上都是保密数据，放在别人的服务器上安全吗？公司政策不允许把敏感数据放在别人的服务器上。估计这是好多用户还不能放开手脚使用AutoCAD WS的重要原因。

 

那这个矛盾有办法解决吗？我的DWG数据一定要放在Autodesk的云存储才能使用AutoCAD WS吗？能不能放在我自己的服务器上呢？其实AutoCAD WS可以存取任何支持WebDAV协议的服务器。常见的比如Dropbox，Box.com,Busszzaw,MobileMe iDisc, Egnyte甚至还可以和AutoCAD易捆绑。我们对数据放在别人服务器上不放心，那我们放在自己的服务器不就可以了吗，只有我假设一个支持WebDAV协议的服务器就可以了。你可以使用Apache或者IIS，或者其他任何支持WebDAV协议的服务器来作为你的数据存储。IIS已经内置的WebDAV协议，我们以Win7上的IIS 7.5为例说一下如何假设自己的WebDAV服务器供AutoCAD WS使用。

 

首先要打开WebDAV协议：

控制面板— Programs and Features – Turn Windows Features on or off, 勾选WebDAV Publishing，如图：

 

具体过程请参考http://wenku.baidu.com/view/395fc3ddad51f01dc281f13b.html 

 

在我们自己的WebDAV协议配置成功后，即可以把自己的DWG数据放在自己的WebDAV服务器上，然后有AutoCAD WS进行存取。当然了，为了让处于外网的AutoCAD WS能访问你的WebDAV服务器，你的WebDAV服务器要从外网能够访问，你需要有自己的域名或者静态IP地址才行。

 

下面登录到AutoCAD WS，在菜单上找到“连接”，即可打开“连接到服务”对话框，点击右边的按钮你可以看到AutoCAD WS预设支持上面我们提到的常见WebDAV服务器。这里你可以填入你自己的WebDAV服务器地址，用户名和密码已经文件夹名称。即可实现从AutoCAD WS访问位于自己服务器上的DWG文件了。

 

你觉得这个想法如何？有什么看法欢迎大家一起讨论。
