前端应聘要准备些什么样子的作品？

Nov 16, 2017

文章背景：来自于其它网站的一次线上交流，当时回答感觉比较粗糙，现重新整理分享出来，同时也是阶段性的总结。

有些人还在疑惑要不要提供作品，怎么样提供，这个在上文中有谈到 ( 我们前端是怎么找到工作的 )，作品是必须要提供的，社会是竞争的，你不提供，别人提供，机会就是这样让出去的。提供一切的作品，一切在线资源的目的是为了更形象方便的证明你某方面的能力，提供更多的资源是为了更全面的了解，通过简短的几十分钟面试，面试官并不能全面的了解你，提供其它信息，有助于面试官通过其它渠道对你进行了解，进而进一步核实你的能力是否与公司岗位要求匹配。简单说，你说你很牛逼，但你得让别人方便的知道你很牛逼，只凭一句话是不行的，U盘拷文件也是不行的，因为你还没有见到面试官，需要的是机会，见到面试官的机会，这需要方方面面的准备，作品是其中最直接的一种方式。

 

1. 证明工作经历的，具备某种解决问题能力的，具备解决某种问题经验的。

公司现有官网，相关产品地址，如果没有就提供代码截图，界面截图，静态页面等等。反正得有一样，证明你曾经有这个经历，在交流时，证明当时担任这个角色的时候做了那些东西，当中遇到了什么困难或问题，你是怎么解决的，等等的，虽然有些回答并不完美，有时候处理并不合适，但这就是成长的痕迹，过于完美，那不就成长，叫电影艺术，况且电影艺术也是强调成长，小人物的心理。星爷的电影感动我们不就是因为他在任何时候都在挣扎，都在试图成长。

 

2. 证明学习能力的，博客，里边应该有若干篇学习新东西的博文。

证明你业余时间，充分利用，并对新技术有自己的见解或看法，即能随波助流，又能剑走偏峰。同时也是记录总结的利器，以后积累经验，工作中迅速寻找解决方案的来源之一。

 

3. 证明对新技术探索能力的，github里边应该有几个现有热门技术的项目。

表示在寻找新技术解决方案的路上，一直保持着一颗敬畏之心，一颗虔诚的心，时刻保持学习或了解新事物的态度或激情。 
岑安作品集

 

4. 证明对其它技术热情的，可以单独弄一个列表的。

实例索引

豪情作品集
像这样的列表，扔掉那些约束技术表现的商业因素，只是纯前端的思想来展现你所表达的东西。

博客无所谓什么皮肤，你的岗位是前端，着重的是css,js能力的体现，所以先随便整一个咯。建议博客园，好处是皮肤可以自己定义，没这个能力的暂时弄一个现成的。像我的这样： 豪情 - 博客园 ，皮肤也有： 黑绿之旅 - 博客模板

简单一句话，你的作品是为你的能力服务，擅长哪个领域就准备哪个领域的作品。还有就是这些作品是一个结果，其实我提倡在享受积累作品的同时，积累最后的结果。这样面试的时候就技术的细节才能聊得起来。不要因为这个结果，而忽略掉这个过程，过程比结果更重要，过程沉淀在你的脑海中，陪伴终身，灵活变化，结果会因客观因素而掉失掉，保持激情，勿忘初心，坚持。

 

总结：

  • 证明工作经历的只要在你简历上写以往公司作品就可以了。
  • 证明学习能力的博客是最直接的方式了。
  • 证明新技术的关注度：Github，微博里边的话题等。
