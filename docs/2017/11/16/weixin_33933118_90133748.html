ASP.NET MVC4.0展望

Nov 16, 2017

最近又回到.NET开发平台上面了，我也一直关注微软技术动向，发现微软想把ASP.NET MVC成为最优秀的构建现代富Web应用程序的平台，所以在开发中也使用了ASP.NET 4.0 MVC 3.0进行开发。在MVC 3.0中的前台UI的Razor视图引擎新语法也比较美观，我一直很讨厌使用尖括号和百分号那种相互交替的前台，当然也挺讨厌控件，特别是ViewState等等。在MVC3.0中微软已经解决了这些，用户控件已经消失，ViewState基本被ViewBag动态对象替代。

 

在MVC3.0中ASP.NET Web页面可以在Web应用程序目录的App_Code文件夹中添加.cshtml或.vbhtml文件，使用Razor语法来编写辅助方法。尽管在ASP.NET MVC项目中也可以添加这些文件，但它们不能访问ASP.NET MVC上下文对象（如ViewContext）或ASP.NET MVC HtmlHelper实例。在ASP.NET MVC 4中，有可能可以编译起来，使用类库的形式进行调用。

 

 今天浏览了一下scottgu的博客，其中还提到了WebAPI的应用，具体地址：http://weblogs.asp.net/scottgu/archive/2012/02/23/asp-net-web-api-part-1.aspx。大家可以看看，在国内很多大公司就有提供API，像腾讯，人人网，以及淘宝等等，我们在调用它们的API时也很好用，以后自己的网站上也可以提供API给别人，当然是有WebService等到也可以实现。不过只要微软提供了，肯定没错，信微软，做.NET得永生......

 

现在很多网站上都集成oAuth验证，在新版本中也会很好的解决问题，以后调用就会比较简单。在C#新版本中异步和同步编程，这个是很好用的，特别做蜘蛛小偷程序中很好用。由于是在公司撑场面，后面又有人说话，思路不清晰，写作都不知道在写些什么，望各位见谅欢迎指导。
