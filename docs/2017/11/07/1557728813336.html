Skype for Business Server 2015系列（一）概述和准备工作

Nov 7, 2017

对于Lync，相信熟悉微软产品的小伙伴都比较了解了，作为微软旗下的即时通讯软件，面向企业用户，全球500强中的70%都在使用Lync。在今年4月份的时候，微软将 Lync 更名为 Skype for Business，作为 Office 2013 四月更新的一部分，正式开始推送。微软称Skype for Business将集合 Skype 和Lync的优势（如Skype界面熟悉感和Lync企业级功能）界面和图标更像Skype，提供了全新的客户端体验。

 

作为服务器端的Skype for Business Server 2015成为Lync Server的替代产品，除了全新的客户端体验，还提供了多项新功能改进，如用户体验、语音和视频支持、移动支持、内部服务器管理、混合解决方案部署与管理等。

今天将为大家分享如何为企业用户部署一套Skype for Business Server 2015。

开始部署 Skype for Business Server 2015 之前，我们需要一些准备工作。

一、服务器准备工作

1、Skype for Business Server 2015 硬件要求

前端服务器、后端服务器、Standard Edition Server 和 持久聊天服务器 的推荐硬件

边缘服务器、独立中介服务器、视频互操作服务器和控制器的推荐硬件

2、Skype for Business Server 2015 操作系统要求

推荐为所有用于 SFB 的新服务器使用 Windows Server 2012 R2。只有在现有服务器已经安装 Lync Server 2013，并且希望对其执行就地升级的情况下才应使用 Windows Server 2008 R2。

除了最新的 Service Pack 外，您还应确保在相关的地方安装以下更新：

对于 Windows Server 2012，应首先安装知识库文章 2858668 提供的更新，然后再进行升级。

https://support.microsoft.com/zh-cn/kb/2858668

如果您拥有 Windows Server 2012 R2，请首先安装知识库文章 2982006 提供的更新，然后再进行升级。

https://support.microsoft.com/zh-cn/kb/2982006

如果您是在 Windows Server 2008 R2 机器上进行升级（参阅上述备注），则您应首先安装知识库文章 2533623 提供的更新。

https://support.microsoft.com/zh-cn/kb/2533623

3、Skype for Business Server 2015数据库要求

注：安装 Skype for Business Server 2015 标准版 时，还会自动安装 SQL Server 2014 Express（64 位版）。

二、操作系统准备工作

1、Skype for Business Server 2015 部署之前安装的软件

运行 Skype for Business Server 2015 的服务器需要安装和配置：

Windows PowerShell 3.0（Windows Server 2012 或 Windows Server 2012 R2已经包含了此功能 ）

Microsoft .NET Framework 4.5（选中并且安装HTTP 激活）

Windows Identity Foundation （服务器管理器--添加角色和功能向导--功能

选中“Windows Identity Foundation 3.5”，单击“下一步”，再单击“安装”）

远程服务器管理工具（服务器管理器--添加角色和功能向导--功能—远程服务器管理工具--角色管理工具 ，选中‘‘’AD DS 和 AD LDS 工具‘’）可选项

2、SFB前端服务器 和 Standard Edition Server 需要的Windows角色和功能

前端服务器 以及所有 Standard Edition Server 需要的 IIS模块：

常见的 HTTP 功能：默认文档、HTTP 错误、静态内容、

运行状况和诊断：HTTP 日志记录、日志记录工具、跟踪、

性能： 静态内容压缩、动态内容压缩、

安全性：请求筛选、客户端证书映射身份验证、Windows 身份验证、

应用程序开发：.NET Extensibility 3.5、.NET Extensibility 4.5、ASP.NET 3.5、ASP.NET 4.5、ISAPI 扩展、ISAPI 筛选器、

管理工具：IIS 管理控制台、IIS 管理脚本和工具

Windows Media Format Runtime

.NET Framework 3.5

Silverlight

三、Skype for Business Server 2015 的环境要求

Active Directory（Windows Server 2012/ R2、Windows Server 2008/R2）

域功能级别（Windows Server 2012/ R2、Windows Server 2008/R2、Windows Server 2003）

域名系统 DNS

证书服务器及证书

文件共享

四、Skype for Business Server 2015实验拓扑：

附：

Skype for Business Server 2015各个服务器功能（企业版为例）

后端服务器

存储在后端服务器数据库中的信息包括状态信息、用户的联系人列表、包含所有当前会议状态相关的永久数据的会议数据，以及会议安排数据。

前端服务器

用于 用户登陆身份验证和注册、IM 功能（包括多方 IM 会议）、Web 会议、PSTN 电话拨入式会议和 A/V 会议、移动功能等

中介服务器（可并置在前端）

提供Skype for Business Server 2015与企业的电话系统集成，集成后，你可以通过Skype for Business 2015拔打和接听电话。

监控和存档服务器（已并置在前端无需单独部署）

用于存档即时消息 (IM) 对话和组会议，以及在呼叫详细信息记录中捕获与文件传输、音频/视频 (A/V) 对话、应用程序共享、远程协助、会议和会议服务器相关的使用信息。

持久聊天服务器

群聊服务器，类似QQ群，功能不够QQ群强大

边缘服务器

让Skype for Business 2015用户在internet上面可以登录，使用即时消息、音频会议、视频会议、文件传输等功能。

反向代理服务器（TMG2010、硬件防火墙等）

让外部用户能够访问内部的Skype for Business、Office Web App、Exchange等服务器；让外部用户能够下载通讯簿、会议内容，移动客户端登录等功能
