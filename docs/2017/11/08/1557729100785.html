Microsoft Azure Site Recovery (2) 配置虚拟机保护

Nov 8, 2017

继上一篇配置完成VMM保护后，登陆到Azure恢复服务，打开保管库受保护的项，可以看到我们VMM服务器中的云已经映射到了保管库，下面打开其中一个云，准备进行保护配置。

 

打开SilverCloud云后，选择配置保护设置。

 

目标选择为Microsoft Azure，然后选择相对应的订阅和存储账户。复制频率可选择30秒、5分钟和15分钟

 

接下来登陆到VMM服务器，打开需要保护的虚拟机的硬件配置并启用Hyper-V Recovery Manager保护。

继续回到Recovery服务，选择映射网络。

 

把VMM中的网络和Azure虚拟网络做一个网络映射。

 

回到Recovery保管库，为虚拟机启用保护。

 

等待第一次复制，时间很漫长。
