JIRA重启服务器后需要重启TOMCAT的解决

Nov 10, 2017

装完JIRA后，每次重启服务器后，都要重启一遍TOMCAT6,实在麻烦，今天终于找到了原因。

原因是JIRA需要使用SQL SERVER,但是SQL SERVER  还没起来，找到原因就好解决了，调整服务的启动顺序或者服务的依赖

在注册表中更新HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tomcat6 服务,在基中添加一个DependOnService多文本键值,把MSSQLSERVER添加进去就行了

 

警告：如果使用注册表编辑器或其他方法错误地修改了注册表，则可能会出现严重问题。这些问题可能需要重新安装操作系统才能解决。Microsoft 不能保证可以解决这些问题。修改注册表需要您自担风险。请确定已经对相应的注册表键值做了相应的备份. 
1. 运行regedt32(不要使用regedit)打开注册表中找到你想要启动的服务. 所有的服务都在HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\ 
如windows search的服务名为Wsearch, 对应的注册表位置就在HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\wsearch 
2. 在下面找到Group这个键值,类型为字符串(REG_SZ).如果没有则表示改服务不属于任何的ServiceGroup,需要添加一个字符串(REG_SZ),命名为”Group”, 双击开打编辑,定义一个组名(不要与原有ServiceGroup重复) 
3. 分别记录下你需要的服务所属的Group的值. 
4. 打开HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\ServiceGroupOrder, 其中的List键值的列表就是group运行的顺序,即是属于这些group的服务运行的顺序.从上往下运行,只需要调整Group的行顺序,就可以达到修改启动顺序的目的.如果是自定义的Group,则添加一行.
