ACT完善与修改测试

Nov 15, 2017

大多数情况下，一次测试运行无法提供足够的信息来回答有关 Web 应用程序的所有问题。
请检查 ACTTrace.log 文件和报告，并确保不存在以下错误：
  • 脚本错误
  • DNS 错误
  • 套接字连接错误
  • HTTP 错误
如有必要，请在重新运行测试之前改正其中的错误。无效的 URL 将导致 Web 服务器返回错误响应代码并关闭 HTTP 连接，降低 Web 应用程序的性能以及测试结果的精确度。
测试运行后，应检查以下性能计数器：
  • Web 服务器的处理器使用率。特别是，如果 Web 服务器处理器使用率未达到 80% 以上，请增加 浏览器同时连接数来增大负载。
  • Application Center Test 客户端的处理器使用率。如果该使用率处于最大值，而 Web 服务器的使用率没有处于最大值，可能无法创建足够的负载以确定服务器的最大容量。
再次运行测试，并记下 RPS 值的更改、首字节响应时间 (TTFB) 和末字节响应时间 (TTLB)。再次检查 Web 服务器处理器的使用率水平，如有必要，请增加负载进行下次测试运行。如果处理器使用率达不到 80% 以上，则瓶颈可能不是处理器速度，请使用性能计数器进行检测。


如果测试运行序列的最大 RPS 值达不到服务器处理器的最小使用率 80%，可能是其他瓶颈限制了性能。如果服务器达不到至少 80% 的处理器使用率，而 ACT 客户端的处理器使用率处于最大状态，则很可能是 ACT 没有为服务器施加足够的负载。
