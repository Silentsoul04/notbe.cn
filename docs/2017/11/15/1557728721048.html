Exchange Server2010系列之八：利用通讯组方便地进行收件人管理

Nov 15, 2017

        通讯组由来已久，早在Exchange5.5的时候就出现了邮件列表和组的概念，但二者之间没有关联，从Exchange2000开始，微软就刻意做了改变，邮件列表和组紧密结合。管理员可以给组分配一个邮件地址，那么这个地址就成为一个邮件列表，给这个列表发信，则组内的所有邮件接收对象包括邮箱，联系人等都能收到。有了通讯组，使得管理员对邮件列表的管理变得简单轻松，使得用户在发送邮件时节省了时间同时减少出错的机率。

        在Exchange Server2010中，分为静态通讯组和动态通讯组两种。所谓静态通讯组是指定其成员需要管理者手动管理，当有成员加入或退出时就需要管理员及时做变更操作。而动态通讯组的成员可以根据管理员设定的条件自动聚合。动态通讯组是已启用了邮件的 Active Directory 组对象，包含 Active Directory 中属性与组筛选器相匹配的所有收件人。如果修改收件人属性以与筛选器匹配，该收件人可能会无意中成为组成员而开始接收发送给组的邮件。管理员定义好筛选条件非常重要，可以降低发生此类问题的机率。

        新建通讯组：

        一般所说通讯组默认就是指静态通讯组，下面我们就通过示例看一下如何新建静态通讯组，可以使用EMC管理控制台或者是EMS命令，在此我们使用EMC图形界面。打开 &ldquo;EMC管理控制台&rdquo;--&ldquo;收件人配置&rdquo;--&ldquo;通讯组&rdquo;，可以右键&ldquo;新建通讯组&rdquo;如下图所示：

        出现&ldquo;新建通讯组&rdquo;向导之后，选择&ldquo;新建组&rdquo;，组类型选择&ldquo;分发&rdquo;，指定组织单位而不使用默认值中，指定&rdquo;uec.com/Branch&rdquo;。此通讯组将创建到Branch组织单位中，此通讯组可以包括AD DS中所有的含有邮件的对象。名称为&ldquo;分支机构&rdquo;，别名为&ldquo;Branch&rdquo;，如下图所示：

 

 

        然后，依次点击&ldquo;新建&rdquo;、&ldquo;完成&rdquo;即可完成静态通讯组的创建，但是此通讯组没有成员， 我们需要手动添加相应的用户。方法是在组名上右键，打开属性，切换到&ldquo;成员&rdquo;选项卡，从列表中选择需要添加到该组的所有用户即可。如下图所示：

        然后，我们依次查看administrator、dufei、xhl、lsj的邮箱，如下图所示：

 

 

注意：如果外网（其他公司）的用户给通讯组发信失败的话，则需要，在组邮箱属性的【邮件流设置】项—>【邮件传递限制】—>【要求所有发件人通过身份验证】是被选中的（该项在建立普通用户邮箱时是不会被选中的，可能微软为了安全性考虑在建立组邮箱时的默认设置是选中的），取消该选项后外网可正常发送邮件至该邮箱组。

 

       新建动态通讯组：

        通过上面的介绍，可以看到静态通讯组的管理还是相当简单的，但是当用户数量较大，而且成员经常发生变化时，管理员的日子就不太好过了。例如员工更换部门、职务变更、离职等，这种情况就可以使用动态通讯组。下面我们就创建一个动态通讯组，成员只包括Branch组织单位中的用户。

        操作方法是：以管理员身份打开EMC，选择&ldquo;收件人配置&rdquo;--&ldquo;通讯组&rdquo;，右键选择&ldquo;新建动态通讯组&rdquo;，如下图所示：

        打开&ldquo;新建动态通讯组&rdquo;向导，指定组织单位而不使用默认值、名称以及别名等参数。如下图所示：

        在筛选器设置界面中，指定只有Branch组织单位中满足条件的收件人才能应用于筛选器，收件人类型我们在些选择&ldquo;拥有Exchange邮箱的用户&rdquo;也就是只让用户邮箱成为该组的成员，然后下一步：

        因为我们是针对Branch组织单位中的所有邮箱，所以在此不必设置条件。然后依次点击&ldquo;下一步&rdquo;、&ldquo;新建&rdquo;、&ldquo;完成&rdquo;即可。下面我们针对此动态通讯组做一个测试，还是用岳雷这个帐号向此地址发一封测试邮件。以判断是不是Branch组织单位中所有收件人都能收到。如下图所示：

 

        然后，我们依次查看Branch组织单位中的用户是否都收到了邮件，在此我们不一一进行截图，仅以dufei为例，同时还需要确保Branch组织单位以外的收件人无法收到此邮件，如下图所示：

        管理员，还可以通过EMC图形界面很方便地查看通讯组的成员信息，方法是：右键打开组的属性，切换到&ldquo;条件&rdquo;选项卡，点击下面的&ldquo;预览&rdquo;，可以很方便地查看当前满足条件的收件人列表。如下图所示：

        以上就是静态通讯组和动态通讯组的创建和基本操作。当然管理员在日常工作中巧用通讯组可以大大减少管理复杂度，借助通讯组实现对成批用户的管理。一般的实现方法是先创建一个通讯组，为通讯组设置相应的规则，规则生效后，向该组中加入用户即可。没有做不到的，只有想不到的，放开去想吧！
