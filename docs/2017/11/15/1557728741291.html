SCOM 2012 SP1---安装SCOM 2012 SP1

Nov 15, 2017

二、安装SCOM 2012 SP1

1.装入SCOM 2012 SP1的安装光盘，点击“安装”：

在选择要安装的功能界面，我们将SCOM管理服务器，报告服务器，管理控制台和Web控制台以及SQL Server 2012数据库全部部署到了一台机器上。所以这里我们将所有功能全部勾选，如图：

SCOM管理服务器提供了代理和数据库之间的集中通信。

操作控制台提供了一个界面，用于执行监视，管理，创作和报告操作。

利用Web控制台功能，能够基于Web访问SCOM用于执行监视活动。

利用报表功能，可以根据从SCOM数据仓库查询得到的数据在SCOM控制台中构建和呈现报表。

选择安装位置，保持默认，如图。

先决条件检查通过后，选择下一步，如图：

在指定安装选项界面，选择“在新管理组中创建第一台管理服务器”，输入管理组名称scom2012，如图。

同意许可条款，如图。

在配置操作数据库界面，输入服务器名称和实例名称并指定SQL的端口，如图。使用本地默认的SQL实例，直接输入服务器名称win2012-2，如果是远程SQL实例，使用“服务器名称\实例名称”的格式。

然后按照同样的方式配置数据仓库数据库，如图。

选择一个SQL server实例用于reporting services，如图。

注意：选择的SQL Server实例事前必须启动SQL Server代理服务，默认代理服务是关闭的！

在指定要用于Web控制台的网站页面，选择默认网站，这里我们不勾选“启用SSL”，如图。备注：如果勾选“启用SSL”确保IIS服务器已经启用SSL功能。

在选择用于Web控制台的身份验证模式页面，选择“使用混合身份验证”，如图。

在配置operations manager账户界面，为SCOM配置管理服务器操作账户，数据读取器账户，数据编写器账户，如图：

选择是否参加客户体验改善计划，如图。

在安装摘要界面，确认安装没有问题后，选择“安装”，如图。

开始安装，如图。

经过一段时间的等待之后，安装完成，如图：

安装完成后，可以在开始菜单看到SCOM 2012的管理工具，如图。

在服务管理器中，可以看到SCOM相关的服务，如图。

可以顺利打开SCOM的管理控制台，如图。
