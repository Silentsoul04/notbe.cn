ESX VMFS文件系统浏览及修复软件部分免费发布

Nov 1, 2017

    因虚拟化技术的快速发展，越来越多的企业开始将系统体系应用于虚拟化。随之而来的，因人为原因、环境原因、设备原因等，也出现了许多基于虚拟化存储的数据灾难。如何在备份失效的前提下，从故障存储中尽最大可能的恢复数据，成了灾难急救的最后一根稻草。

 

近日，北亚数据恢复中心虚拟化数据恢复研究室发行了两款新软件，一款用于在Windows 平台上直接浏览和导出VMFS 文件系统上的数据，名称为Formbyte Explorer For ESX；另一款用于从损坏的ESX Server 存储中恢复丢失数据，名称为Formbyte Recovery For ESX。

 

近几年，北亚数据恢复中心接手了大量基于虚拟化的数据恢复案例，这当中，绝大多数是关于ESX SERVER的(原因是VMWARE的高市场占有率)。 一些典型的灾难如:vmdk文件误删除、STORAGE MAPPING出错导致的VMFS卷破坏、VMFS存储突发性异常等。为了攻克这些难题，3年前，北亚数据恢复中心成立虚拟化数据恢复研究室，着力于虚拟化领域的数据灾难研究。经过3年来的发展，基于大量案例的研究，也基于大量的虚拟化数据灾难急救需求，研究室决定在原有内部软件的基础上改良并向外推出上述2款软件。

 

Formbyte Explorer For ESX 和 Formbyte Recovery For ESX 这两款软件都支持 ESX(i) Server 3.x - 4.x 的数据存储，对ESX(i) SERVER 5.x 系统新推出的 VMFS 5.x文件系统的浏览及恢复已在内测中，软件下一版本将支持对VMFS 5.x 文件系统的浏览和数据恢复。

 

Formbyte Explorer For ESX 使用方法：

1．先添加数据源。

数据源可以是一块磁盘或一个镜像文件，其中包含ESX的数据存储。

 

 

2．软件会自动列出原来ESX Server 数据存储的信息，如VMFS版本、存储名、卷容量等。然后就可以直接对原来 ESX Server 数据存储中的文件进行恢复操作了。

 

 

 

 

Formbyte Recovery For ESX 使用方法：

Recovery 版与 Explorer 版的区别在于 Explorer 版只能浏览恢复完好或轻微损坏的 VMFS 文件系统。而Recovery 版能恢复如损坏、删除/重建数据存储、删除虚拟机/文件 等灾难性的数据存储。

Recovery 版与 Explorer 版的使用方法差不多，Recovery 版比Explorer 版多了 灾难恢复扫描功能，多了 VMFS 卷搜索功能。

北亚数据恢复中心（http://www.datahf.net） 多年来一直在潜力研究各种高难度的数据灾难恢复方法，领域涉及Windows、LINUX、UNIX及第三方嵌入式系统。在文件系统方面，成功攻克了如NTFS文件系统的碎片流恢复、LINUX 下删除文件的恢复、UNIX小型机文件系统灾难恢复等的世界级难题；在数据库领域，针对ORACLE、MS SQL Server、MYSQL、SYBASE等数据库修复，已经形成了一整套完善的解决方案。到最新的关于 ESX Server 的恢复研发， 北亚数据恢复中心在技术实力一直走在国内数据恢复界的最前沿。
