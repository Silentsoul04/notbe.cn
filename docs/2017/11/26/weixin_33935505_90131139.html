Android + eclipse +ADT安装完全教程

Nov 26, 2017

首先，是要下载android SDK，在http://developer.android.com/sdk/index.html这个页面选择操作系统，这有windows，mac OS,LINUX三个版本的SDK供下载，现在最新版本是10月更新的android-sdk_r07-windows.zip，嗯，这个网站不知道为什么让天朝GWF给墙了，最先我是找了个翻（河蟹啊）墙软件进去下载的，后面才知道，原来还有一个网站，不知道是不是Google android在国内的网站，反正上面内容跟http://developer.android.com/sdk/index.html完全一样的啦。网址是：http://androidappdocs.appspot.com/index.html

http://dl.google.com/android/android-sdk_r07-windows.zip这也可以直接下载。

 

下载之后，解压，出现android-sdk-windows文件夹，进入点击那个机器人图标，也就是SDK Manager.exe，结果果断的弹出了Failed to fetch URL http://dl-ssl.google.com/android/repository/repository.xml, reason: dl-ssl.google.com这个错误，不能连接上这个xml http://dl-ssl.google.com/android/repository/repository.xml,，网上寻求答案，翻遍了网络各位前辈们的解决办法，像什么在Setting里面设置代理服务器：10.159.192.62，端口8080；选中settingforce https。。to http,就是强制将https://...转换成http://..； 之类的各种方法，但每次安装总是仍旧是哪个万恶的结果，数十次。。。我无语了

 

后面果断的按网上的一个方法，直接进入http://dl-ssl.google.com/android/repository/repository.xml,然后在xml中逐个寻找非LINUX,非MAC OS系统的文件，在迅雷里面直接下载下面文件并按下面方法安装：

因为之前我们解压的那个android-sdk-windows文件夹，里面有几个文件夹，下面提到的文件夹，android-sdk-windows里面没有的自己按名字创建一个，ok

以下两个是实例文件，直接解压到samples文件夹：

http://dl-ssl.google.com/android/repository/samples-2.1_r01-linux.zip

http://dl-ssl.google.com/android/repository/samples-2.2_r01-linux.zip

下面的是android的各个版本，因为都可能用到的（当然你也可以将低版本的扔掉），全部解压到platforms里面：

 

https://dl-ssl.google.com/android/repository/android-2.2_r02-windows.zip

https://dl-ssl.google.com/android/repository/android-2.0.1_r01-windows.zip

https://dl-ssl.google.com/android/repository/android-2.0_r01-windows.zip

https://dl-ssl.google.com/android/repository/android-2.1_r02-windows.zip

https://dl-ssl.google.com/android/repository/android-1.6_r03-windows.zip

https://dl-ssl.google.com/android/repository/android-1.5_r04-windows.zip

以下文件是各个API版本，直接解压到add-ons：

https://dl-ssl.google.com/android/repository/google_apis-8_r02.zip

https://dl-ssl.google.com/android/repository/google_apis-7_r01.zip

https://dl-ssl.google.com/android/repository/google_apis-6_r01.zip

https://dl-ssl.google.com/android/repository/google_apis-5_r01.zip

https://dl-ssl.google.com/android/repository/google_apis-4_r02.zip

https://dl-ssl.google.com/android/repository/google_apis-3-r03.zip

 

下面这个是用户手册，解压到docs文件夹：

 

http://dl-ssl.google.com/android/repository/docs-2.2_r01-linux.zip

下面这个文件解压，到tools文件夹：

https://dl-ssl.google.com/android/repository/tools_r07-windows.zip

 

这是证书文件：解压到market_licensing：https://dl-ssl.google.com/android/repository/market_licensing-r01.zip

这是usb驱动文件，解压到usb_driver文件夹：

https://dl-ssl.google.com/android/repository/usb_driver_r03-windows.zip

 

记住，因为zip文件直接解压出来是一个文件夹，里面许多文件，前面的是直接解压到目标文件夹即可，后面的几个解压到这些文件夹的  docs,tools,usb_driver, market_licensing，是要将直接解压后的那个外面文件夹去掉，比如说，USB驱动文件zip解压出来是这个文件夹：usb_driver_r03-windows，要将里面的文件全拷出来然后放到usb_driver里面。不然读不出来；

安装完成之后  将（因为我是安装在E盘下的 所以是这样）E:\android-sdk-windows\tools添加到环境变量path中，注意正确添加，两个路径之间有分号；

 

然后在Eclipse里面，安装ADT:

我这是Eclipse3.5.2.的  其他版本的也都差不多的了

HelpàInstall Softwareà进入Install面板，在那个work with后面，点击add，进入Add Site

面板 name随便写 自己认得就行，Location里面填写：https://dl-ssl.google.com/android/eclipse  ok，Eclipse会自动找到要安装的包，或者如果你先下载好了ADT 这个地址下载： http://dl-ssl.google.com/android/ADT-0.9.9.zip，可以直接点击Archive..找到你下载的ADT文件，按流程就可以安装了，安装好ADT之后，重启Eclipse，

 

点击windowàpreference 看到有个Android了，单击它，在右边 点Browse选择你android所在的本地目标地址 Apply 。。ok。

点击window—》Android SDK and AVD manager 打开面板，左边选Installed Package就可以看见你所安装的东西了，  在Available Packages里面可以选中右边框中的复选框，点击refresh，看看还有什么需要安装的。。。（其实，就在这，如果你之前是先安装Eclipse+ADT的话，你就可以不用困扰于之前安装不上SDK了，直接在这可以在线安装，不知道为什么，点击下载的SDK安装文件SDK Manager.exe连接不上，但在这却可行。。。当然，在线安装远远不足直接下载安装来的快，我这是不到20k的速度。。。总共大约要下载1G）
