使用可重启AD DS 整理活动目录数据库

在所有运行Windows Server 2008的域控制器上可重启的AD DS是默认有效的。在使用这个功能上没有任何功能级或者其它必须具备的要求。

这项特性提供了什么新功能？

在Windows Server 2000或者Windows Server 2003的活动目录中，脱机数据库整理需要将域控制器重启至目录服务恢复模式。应用安全更新也经常要求重新启动域控制器。

然而在Windows Server 2008中，管理员能够停止或者重启AD DS。这使得能够更快速的执行AD DS脱机操作成为可能。

1. 如图12-4所示，在essDC上，点击“开始”à“程序”à“管理工具”à“服务”。

2. 如图12-5所示，打开服务管理器，右击“Active Directory Domain Services”，点击“停止”。

图 12-4 打开服务管理工具 图 12-5 停止活动目录服务

3. 如图12-6所示，在出现的其他服务对话框，点击“是”。

4. 如图12-7所示，点击“开始”à“运行”，输入CMD打开命令提示符，输入ntdsutil回车。

5. 如图12-7所示，在ntdsutil：提示符下输入file，回车。

6. 如图12-7所示，在file maintenance:提示符下输入 info，回车。可以看到活动目录数据库和日志文件的位置。

图 12-6 停止相关服务 图 12-7查看数据库和日志文件

7. 如图12-8所示，在file maintenance：提示符下输入compact to c:\temp，回车。压缩数据库到c:\temp文件夹下。输入quit，quit推出ntdsutil。

8. 如图12-9所示，在命令提示下copy c:\temp\ntds.dit c:\windows\ntds\ntds.dit 回车，出现是否覆盖提示，输入Y，回车。

9. 如图12-9所示，在命令提示符下输入del c:\windows\ntds\*.log，删除日志文件。

图 12-8 压缩数据库文件 图 12-9 覆盖数据库文件

10. 如图12-10所示，在服务工具，右击Active Directory Domain Services服务，点击“启动”，相关服务也会自动启动。

图 12-10 启动活动目录服务
