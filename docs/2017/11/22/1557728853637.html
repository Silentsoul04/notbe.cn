重要提示：

在安装 Microsoft Lync Server 2010 群聊之前，必须卸载旧版 Microsoft Lync Server 2010 核心组件或 Microsoft Office Communications Server，以及旧版 Microsoft 统一通信托管 API (UCMA)。有关卸载核心组件的详细信息，请参阅“Microsoft Office Communications Server 2007 R2 累计更新的描述，核心组件：2010 年 9 月”，网址为 http://go.microsoft.com/fwlink/?linkid=209091&clcid=0x804。

使用两个向导安装 Lync Server 2010 群聊：群聊服务器安装向导和服务器配置向导。群聊服务器安装向导在计算机上安装服务器和服务。服务器配置向导（在完成群聊服务器向导后自动打开）准备服务器和服务以供在您的环境中使用。

要安装群聊服务器，对于运行查找服务、频道服务和 Web 服务的每台服务器，您都必须拥有由 Lync Server 2010 内部服务器所使用的同一个证书颁发机构 (CA) 所颁发的证书。在开始群聊服务器安装之前，确保拥有所需的证书，尤其是使用外部 CA 时。有关证书的详细信息，请参阅为群聊服务器获取证书。

安装初始的群聊服务器实例之后，可以安装其他群聊服务器实例以创建群聊服务器池，从而实现可伸缩性和故障转移。

重要提示：

我们建议您在同一 Lync Server 受信任的应用程序池中安装其他所有的群聊服务器实例。在此环境中，所有群聊服务器的证书必须使用相同的池完全限定的域名 (FQDN)（而不是群聊服务器 FQDN）。您可为证书的使用者备用名 (SAN) 称制作任意条目。建议您为群聊服务器池中的所有服务器实例使用以下证书：SN = “Group Chat Server Pool Name”; SAN = <any entry>。

这些服务器实例必须安装在与原始的群聊服务器安装不同的计算机上。如果要安装合规性服务，可以使用相同的向导，但必须将其安装在一台单独的计算机上。

有关可伸缩性的详细信息，请参阅规划文档中的容量规划。

注意：

完成安装后，建议您更改频道服务帐户、Web 服务帐户和合规性服务帐户的数据库角色成员身份，从 db_owner 分别改为 ChannelServer、WebServer 和 ComplianceServer。这些新角色是在安装过程中创建的。有关分配数据库角色的详细信息，请参阅设置 SQL Server 帐户和权限。

8.1.1-安装群聊服务器

以下过程介绍如何将 Microsoft Lync Server 2010 群聊安装为单服务器拓扑，或池拓扑中额外的服务器实例。服务器实例包括查找服务、频道服务和 Web 服务。如果要安装合规性服务，则必须使用用于安装的同一向导将其安装在其他计算机上。

重要提示：

尽管安装向导可让您选择在同一台服务器上同时安装群聊服务器和合规性服务，但群聊服务器不支持在同一台计算机上同时承载这两者。请确保在安装过程中选择群聊服务器或合规性服务，而不是同时选择两者。

如果要另行安装一个Lync Server 2010 群聊实例作为群聊服务器池的一部分，则会使用服务器池中已确定的设置，且部分控件不可用于所有群聊服务器均使用的设置。共享设置包括：

  • Microsoft Lync Server 2010无控件可用。
  • 查找服务仅“密码”、“确认密码”和“池通信端口”框可用。

要安装群聊服务器，计算机上必须已安装以下软件：

  • Microsoft .NET Framework 3.5 SP1，用于运行所有群聊服务器服务，因为这些服务是使用 .NET 编写的

  • Internet Information Services (IIS)，用于承载 Web 服务,以将文件发布到聊天室

重要提示：

Windows Server 2008 用户必须启用 IIS 6.0 兼容功能。
  • ASP.NET 2.0，用于构建动态网站、Web 应用程序和 XML Web 服务

注意：

ASP.NET 2.0 是一个 Web 应用程序，该应用程序是 .NET Framework 的一部分。
  • 消息队列，如果已部署合规性服务，则由群聊服务器合规性服务使用。

如果未安装以下应用程序，则将自动安装这些应用程序：

  • Lync Server 2010 核心组件
  • Visual C ++ 2008 可再发行运行时 9.0.2102 版
  • Microsoft 统一通信托管 API (UCMA) 3.0 可再发行软件包

注意：

有关卸载 Lync Server 核心组件的详细信息，请参阅 Microsoft 知识库文章 2407349“Office Communications Server 2007 R2 核心组件累计更新的说明：2010 年 9 月”，网址为 http://go.microsoft.com/fwlink/?LinkId=209091。

安装群聊服务器

 

1. 使用先前设置的帐户，登录到要安装群聊服务器的计算机。

注意：

有关详细信息，请参阅设置群聊服务器帐户和权限。

2. 运行 ServerSetup.exe。

3. 如果未安装 Visual Studio 2008 C++ 运行时的相应版本，请在指示必须安装该软件的对话框中单击“是”。

4. 如果未安装的 UCMA 3.0 可再发行软件包的相应版本，请在指示必须安装该软件的对话框中单击“是”。

5. 在“欢迎”页上，单击“下一步”。

6. 在许可协议页上查看许可协议，单击“我接受许可协议中的条款”以继续，然后单击“下一步”。

7. 在“客户信息”页上指定用户名和组织名称，然后单击“下一步”。

8. 在“安装位置”页上，验证默认位置是否为 <驱动器号>:\Program Files\Microsoft Lync Server 2010\Group Chat Server\，然后单击“下一步”。

9. 在“选择功能”页上单击“合规性服务”左侧的下拉列表框，然后单击红色 X 以指定计算机上将不安装合规性服务。

注意：

在计算机上将仅安装群聊服务器，其中包括查找服务、频道服务和 Web 服务。不能在同一台计算机上同时安装群聊服务器和合规性服务。

10. 单击“下一步”。

11. 在“确认安装”页上，单击“下一步”。此时将打开“服务器配置”向导，指示安装了查找服务、频道服务和 Web 服务。

12. 完成“服务器配置”向导。

注意：

进行此过程的下一步之前，必须先完成“服务器配置”向导。有关详细信息，请参阅配置群聊服务器。

13. 完成“服务器配置”向导之后，请在“安装完成”页上单击“关闭”。

注意：

此处指定的超级用户将成为此安装的初始超级用户。以后可以使用群聊管理工具添加更多超级用户。安装程序的用户名自动显示在“查找和添加超级用户”框中。要向“超级用户”页添加此用户名，请单击“添加”。要向“超级用户”页添加其他用户，请指定 SIP 统一资源标识符 (URI) 或用户名，然后单击“添加”。

2. 在“端口”框中，选中“自动检测端口”复选框，或者指定运行 Lync Server 2010 的计算机用于相互 TLS (MTLS) 连接的端口。

注意：

计算机上安装的查找服务和频道服务必须使用不同的池通信端口。例如，如果查找服务的池通信端口为 8011，则该计算机上频道服务的池通信端口不能为 8011。

4. 如果“站点名称”框为空，请打开 Lync Server 命令行管理程序，然后运行 Get-CsSite cmdlet 以获取站点名称。

6. 在“服务侦听端口”中，指定用于侦听入站客户端连接的端口。此端口的默认值为 5041。安装程序将使用此端口为频道服务创建受信任的应用程序池、受信任的应用程序计算机和受信任的应用程序。

8.2.3-手动配置群聊管理工具连接

上一次修改主题： 2011-02-02

如果尚未将域名系统 (DNS) 配置为在连接时自动返回相应的 SRV 记录，请使用以下过程手动输入帐户配置信息。

使用手动配置的帐户进行连接

 

1. 打开 Microsoft Lync Server 2010 群聊管理工具，方法是单击“开始”，再单击“所有程序”，指向“Microsoft Lync Server 2010”，然后单击“Microsoft Lync Server 2010 管理工具”。

2. 在“群聊管理工具登录”对话框的“帐户”框中，单击“编辑帐户”。

3. 在“编辑帐户”对话框中，单击“添加”，向“帐户”列表添加新帐户。

4. 在“显示名称”框中，键入新帐户配置设置的名称。

注意：

新配置的名称将添加到“群聊管理工具登录”对话框的“帐户”框中，以供将来登录时使用。

5. 单击“登录设置”选项卡。

6. （可选）选中“使用 Windows 凭据自动登录”复选框以使用单一登录，此方式将自动从 Windows 操作系统中加载密码。

7. 在“主机”框中，指定运行 Microsoft Lync Server 2010 的服务器的完全限定域名 (FQDN)。

8. 在“聊天室域”框中，键入查找服务器的 SIP 统一资源标识符 (URI) 的域部分（例如，如果 SIP URI 是 kazuto@litwareinc.com，则键入 litwareinc.com）。

9. 执行下列操作之一：

o 如果使用查找服务帐户的默认 SIP URI（例如，Chat@litwareinc.com），则选中“使用默认服务器地址”复选框。

o 否则，请指定查找服务器的 SIP URI（在安装 Microsoft Lync Server 2010 群聊时指定）。

10. 单击“Active Directory”选项卡。

11. 在“全局编录”下执行下列操作之一：

o 单击“自动连接配置”。

o 单击“手动连接配置”，并在“主机”中键入全局编录服务器的 FQDN。全局编录是 Active Directory 域服务 (AD DS) 中的可搜索索引，使用户可以定位网络对象，而无需知道这些对象的域位置。

12. 选中“仅使用安全连接”复选框，从而使用安全套接字层 (SSL) 连接来连接到 AD DS。

13. 如果频道服务帐户无权访问 AD DS，则选中“连接身份”复选框，键入有权访问 AD DS 的频道服务帐户的名称和密码，然后单击“确定”。

14. 在“群聊管理工具登录”对话框中键入用户名。
