出现了这个错误顿时晕了，因为刚接触birt报表。不知道从那里入手，后经过百度和请教人，总结了例如以下的解决方法：

因为公司创建birt报表的版本号较低为2.6.2，我如今用的是4.4.2版本号。因为版本号不一致引起的错误。注意报表文件的开头：

<report xmlns="http://www.eclipse.org/birt/2005/design" version="3.2.23" id="1">
    <property name="comments">版权全部 (c) 2006 <<此处是您公司的名称>></property>
    <property name="createdBy">Eclipse BIRT Designer Version 4.4.2.v201410272105 Build <4.4.2.v20150217-1805></property>
report节点的属性version的值为：3.2.23。此时我用的birt的版本号为4.4.2（version相应为3.2.23），而公司用的是2.6.2（version相应为3.2.22）。仅仅须要将version的值改动为低版本号即可（3.2.22）即可，只是建议改动下name="createBy"的property节点的文本值（这个修不改动都不会报错的），改为曾经的文本值（改动birt报表文件之前建议将这个值先保存下来）。避免发生混淆。改动后例如以下： 

<?xml version="1.0" encoding="UTF-8"?>
<report xmlns="http://www.eclipse.org/birt/2005/design" version="3.2.22" id="1">
    <property name="comments">版权全部 (c) 2006 <<此处是您公司的名称>></property>
    <property name="createdBy">Eclipse BIRT Designer Version 2.6.2.r262_v20110209 Build <2.6.2.v20110214-1523></property>
这些改动应该依据你自己的实际情况进行改动，上面仅仅是以我自己的为例。
