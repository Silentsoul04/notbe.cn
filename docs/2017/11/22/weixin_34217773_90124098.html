centos下一个bash: XXX: command not found解决方案

Nov 22, 2017

最近想centos通过做Android工程建设。配置jdk和Android sdk后，也同时/etc/profile将java和Android环境变量配置成，但它不能像windows 在相同，直接使用android update project命令。

出现情况例如以下:bash: android: command not found

此时，我们能够使用ln命令将android 加入到/usr/bin文件夹下，具体解决方法例如以下:

1.使用cd /usr/bin 进入到该文件夹

2.使用ln -s -f  android sdk安装文件夹/tools/android命令，将android链接到该文件夹上。

3.完毕。此时我们就能够在随意文件夹使用android命令啦。


所以，假设出现bash: ant: command not found等情况，均能够使用此方法尝试解决。

最后。附上linux ln命令的说明：ln是linux中又一个很重要命令，它的功能是为某一个文件在另外一个位置建立一个同步的链接。

语法:

1.将某个文件链接到一个文件上  ln [-f | -n] [-s] SourceFile [TargetFile]

2.将一个或者多个文件链接到一个文件夹上ln [-f | -n] [-s] SourceFile ... TargetDirectory
