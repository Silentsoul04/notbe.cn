关于ubuntu下qt编译显示Cannot connect creator comm socket /tmp/qt_temp.xxx/stub-socket的解决的方法...

Nov 22, 2017

今天在ubuntu下安装了qtcreator，准备測试一下能否用。果然一測试就出问题了，简单编写后F5编译在gnome-terminal中出现

Cannot connect creator comm socket /tmp/qt_temp.u14973/stub-socket: No such file or directory  

于是查了一下。知道qt不能用gnome-terminal运行，须要使用x-team。于是想到了解决方法：

找到：工具（Tools）->环境（environment）->系统(System)->终端(Terminal)。把x-terminal-emulator-e改动为 /usr/bin/xterm -e

正常情况下问题就已经攻克了。只是非常不幸，我运气不好，F5后又出现了error：

ptrace 不同意的操作

百度了一下找到了解决方法：

$ sudo vi /etc/sysctl.d/10-ptrace.conf
$ kernel.yama.ptrace_scope = 0
最后保存重新启动就可以，假设不会使用vim就 

$sudo gedit /etc/sysctl.d/10-ptrace.conf

把最后一行改动为 kernel.yama.ptrace_scope = 0保存重新启动就可以。
