Exchange Server2010系列之九：轻松配置脱机通讯簿

Nov 14, 2017

    Exchange Server 2010 中的脱机通讯簿又称为OAB（Offline Address Book  是指已下载的通讯簿副本，以便 Outlook 用户在与服务器断开连接时可以访问其中所包含的信息。Exchange 管理员可以选择脱机工作的用户能够使用哪些通讯簿，还可以配置通讯簿的分发方法，早期OAB版本只能通过公共文件夹的形式进行更新，只支持Outlook 2003 及以前的版本。Exchange 2007的发布之后，OAB基于网络的分布模型设计，基于Web的分发方式更为有效地利用带宽，在使用Outlook 2007或Outlook2010的客户端时，可以来通过WEB方式来更新OAB，为保留与Exchange2003的兼容同时支持公共文件夹方式的发布。

    新建脱机通讯簿

    一个脱机通讯簿正常使用要经过三个步骤：在邮箱服务器上创建----在客户端访问服务器上发布-----客户端Outlook下载即可。Exchange Server2010安装完成后，生成一个“默认脱机通讯簿”，一般情况下使用此此通讯簿即可。但在一般特殊环境中需要新建脱机通讯簿，例如只显示部分用户的地址信息。新建脱机通讯簿的方法是：在邮箱服务器上打开Exchange管理控制台，“组织配置”---“邮箱”，切换到“脱机通讯簿”选项卡上，右键“新建脱机通讯簿”，如下图所示：

    点击打开“新建脱机通讯簿”，出现简介对话框，如下图所示：

    在此界面中，输入此脱机通讯簿的名称，生成服务器也就是目标邮箱服务器，以及此通讯簿所使用的地址列表，在此我们使用“包括以下地址列表”，并添加我们前面已经创建的Branch分支机构这个地址列表，这样的话此通机通讯簿只包括此分支机构地址列表的信息。如果选择“包括默认全局地址列表”，则与默认脱机通讯簿完全相同。

    再往下就是分发点配置界面，分发点的目的是让客户端能够下载脱机通讯簿。其中两种分发方式，在此选择“启用WEB式分发”，添加，选择相应的虚拟目标名称以及CAS服务器名称。如果组织中有Outlook2003客户端，则还需要选中“启用公用文件夹分发”。然后根据向导，依次点击“新建”、“完成”即可。

    更改默认脱机通讯簿

   因此在安装邮箱服务器角色时，系统已经创建一个名为“默认脱机通讯簿”的基于 Web 的默认脱机通讯簿 (OAB)。下面我们需要更改默认脱机通讯簿，将 新建的脱机通讯簿设置为默认 OAB。这一新的默认 OAB 将与所有新创建的邮箱数据库相关联。在Exchange组织中只能有一个默认 OAB。如果仅仅 删除默认 OAB，Exchange 不会将新的 OAB 自动指定为默认 OAB。管理员必须手动将另一个 OAB 指定为默认 OAB。方法也非常简单，在新建的OAB上点击右键“设置为默认值”即可，如下图所示：

    此时，会弹出下面的提示：

     在此选择“是”，将我们新建的OAB设置为所有新邮箱数据库的脱机通讯簿。

    更新OAB

    接下来需要在CAS服务器上手动更新OAB信息，相当于更新此数据库的内容。在生产环境中也可以使用更新计划，默认是每天5：00。手动更新的方法是在“北京分支机构员工”OAB上右键，在弹出的快捷菜单中选择“更新”命令，然后选择“是”如下图所示：

    如果在更新的过程报以下错误，”无法对对象北京分支机构员工执行更新操作“，则有可能是因为Exchange系统助理服务没有启动，只要去系统服务中找到Exchange System Attendent服务启动即可。

     邮箱数据库配置

    指定某邮箱数据库所使用的脱机通讯簿，方法是打开指定的邮箱数据库属性，切换到”客户端设置“选项卡，针对”脱机通讯簿“进行设置，如下图所示：

   以上操作都属于在邮箱服务器端的生成OAB的操作和配置，接下来我们需要在CAS服务器上配置脱机通讯簿的分发。

    脱机通讯簿分发

    此过程一般分为两步，第一步是里在Exchange中查看脱机通讯簿的更新时间间隔和URL地址是否正确，方法是在CAS服务器上打开“服务器配置”---”客户端访问“切换到”脱机通讯簿分发“选项卡，打开OAB的属性。如下图所示：

    打开”属性“后可以看到OAB更新间隔是480分钟，可以根据需要进行设置，切换到URL选项卡，查看内部网络和外部网络所使用的URL是否正确。   

    第二步：IIS中关于OAB虚拟目录的配置，我们打开IIS默认站点中的OAB虚拟目录，因为一般企业可能会对默认站点设置HTTP重定向，一旦进行了默认站点的HTTP重定向之后，其下的子目录都将继承此设置，针对OAB来说，我们需要取消重定向设置。如果，不需要客户端通过HTTPS访问，则可以取消SSL设置，如下图所示：

    最后，我们还需要在IIS服务器的OAB虚拟目录中确认”IIS_IUSRS“组具备有”读取“、”读取和运行“、”列出文件夹“权限，方法是右键打开OAB虚拟目录中的”编辑权限“，切换到”安全“选项卡，可点击”高级“进行查看，一般无须更改。

    管理员还可以通过在”OAB”上右键浏览，打开OAB文件夹,查看其中的文件权限，请检查IIS_IUSRS组用户对此文件夹有”读取“、”读取和运行“、”列出文件夹“权限。

同时修改OAB虚拟目录中web.config权限，添加authenticated users组读取和执行权限。如下图所示：

     此外,我们还需要确认DNS中是否有关于Autodiscover的A记录，如果没有则需要创建，不然客户端下载时，会长时间处于”正在处理状态“。创建A记录的方法如下图所示：

    客户端可以测试自动发现服务，方法是启动Outlook后，按住”CTRL“再右键点击通知区域上的Outlook图标，在弹出的快捷菜单中选择”测试电子邮件自动配置“命令，自动发现服务自动输入当前用户的名称，输入密码，仅选择”使用自动发现“后，单击”测试“，即可显示测试信息，从中可以看到OAB的信息，切换到日志选项卡，可以看到成功的字样。

    客户端下载

    客户端此时应该就可以正常下载OAB的信息了，方法是打开Outlook2010，点击文件菜单---信息---帐户设置----下载通讯簿即可。然后在通讯簿中即显示新建脱机通讯簿所对应的地址列表 。如下图所示：

    稍等片刻，即可完成下载。客户可以根据需要使用此OAB中所包括的地址列表，如下图所示：

    如果因为某个员工离职或者是长期休假，也可以在OAB中禁止此用户显示，操作方法也非常简单，打开此用户邮箱，切换到“常规”选项卡，勾选中“不显示在Exchange地址列表中”即可，在此就不做演示了。祝大家好运！
