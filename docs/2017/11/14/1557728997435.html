Lync Server 2013群聊天室创建和简单测试

Nov 14, 2017






在上一篇文章中，我们部署好了Lync Server 2013的群聊服务器，但没有来得及给大家分享搭建之后的效果，而今天我们就来看看Lync Server 2013中的群聊服务器是个什么样的效果。

首先最直观的效果就是我们登录Lync 2013 Preview后，会发现多了一个选项卡，那就是“群”（官方并没有这样定义，但我个人觉得这样很好理解）。我们切换到群界面可以看到下方是三个分类，分别是关注、成员、以及新的，这个意思与我们现在的微博比较相似，当然默认的这里也是空空如也的，我们单击右侧的“+”就可以展开对群的管理，在客户端中可以进行添加、创建（需要有创建的权限才能够打开创建的页面）、创建主题订阅、搜索群的聊天记录以及群的选项。

这里我们使用一个拥有群创建权限的用户登录，然后选择创建群。

如果正常的话，就会打开Lync Server 2013群的管理页面，这里需要登录，登录之后可以创建群，功能看起来比较单一，似乎还在完善中。

单击创建之后，内容就很多了，我们可以进行各种信息的配置，比如群名称，群介绍、成员、管理者。并且可以设置群的类型，可以为公开的，也就是所有用户都可以看到这个群，因为他们在登录的时候会在群选项卡的“新的”中看到这个群；也可以设置为非公开的，创建之后用户不会看到，但可以搜索群名称找到我们的群；还有就是秘密的，只有成员才能找到和使用群。

在下方还可以设置插件，是可以进行选择，但具体是怎样的形式，现在还不得而知。右边可以设置是否从目录继承邀请设置。设置成可以邀请的话，当我们在这里添加成员后，成员就可以在他们的客户端上看到这个群信息。一切搞定后单击下方的创建即可完成群的创建。

创建好后，我们就可以在My Rooms中看到我们创建的群了，鼠标放在上面可以选择编辑等操作。

我们回到客户端，可以发现这时我们的群选项卡旁多了一个“1”，表示有一个新的群被创建了。我们切换到群界面就可以看到这个群了，并且可以看到群的信息，也就是我们所说的公告。当然相比QQ，似乎还是不够本地化，至少不符合我们的习惯、方式。鼠标放到群上可以选择关注、打开、管理等操作，但管理仍然是在网页上完成的，而不是在客户端中。

打开群后的界面与Lync 2013中一般的对话框没有大的区别，只是我们可以单击下方的联系人图标，就可以展开当前在群中的联系人。我个人觉得唯一有一点遗憾的就是不能在任何时候都将群成员显示在这里，这里只会显示当前在群中的，即使联系人在线但没有进入群，这里也是不会显示的。所以还是没办法直观的看到整个群到底有多少成员，除非大家全部在线并且进入了群，可能这也是习惯的差异吧。所以微软才把群成为房间，或聊天室，而不是真正意义上的群。似乎微软觉得进来才应该显示出来，这才符合“Room”的说法。

我们注销当前用户，用另外一个用户登录并打开群，会发现前面的聊天记录仍然在，并且可以看到具体的时间和日期。

我们在另外一台电脑中登录，并且加入到群，可以看到同时两个用户在，可以相互查看对方的联系人卡片。

唯一有一点不清楚的是文件上传在哪里？这个我真没找到，有兴趣的朋友可以研究一下。
