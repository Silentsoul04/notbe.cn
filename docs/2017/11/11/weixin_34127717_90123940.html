关于Solaris 11发布而引发的一些回忆

Nov 11, 2017


    初次接触Solaris是在08年,当时sun公司还健在.当时sun在各个大学里面搞open活动,印象中当时成立了个open developer 组织,有专门的网站,用来做宣传.宿舍里面的一个同学参加sun在学校里面搞的某次活动拿到了一些纪念品和一张光盘.光盘上就是opensolaris,上面带了sun studio和netbeans.据我那同学说,上面的netbeans启动速度蛮快.另外,我一项对这些IT相关的活动比较感兴趣,就开始关注Solaris了. 
    opensolaris是Solaris的开源版本.按我的理解,其实opensolaris是sun公司把一些包含新特性的Solaris发布出来供大家使用,在得到用户反馈和功能改进后,再把新特性加入到Solaris中.所以从普通使用者角度来看,opensolaris和Solaris用起来差不了太多的.反正俺们又不怎么用到它的所谓的高级的新的特性. 
    后面我也去那个open developer网站上申请了一张opensolaris光盘,很可惜,它就跟sun一样,没有撑到2010年.在去年我想制作一个opensolaris的vmware虚拟机时,发现那张opensolaris光盘已经无法读取出数据了... 
    09年毕业后,本来把Solaris都丢到记忆的某个角落了.但因为工作原因,我又接触Solaris了.我所参与的那个给移动做的项目,它所运行的现场的操作系统是Solaris.然后我开始重新去面对solaris了. 
    不得不说,solaris真的是个相当稳定的操作系统,也是一个不太好维护的操作系统.说它稳定,因为我测试机上面安装的solaris系统曾经连续好几个月没去关过机,上面跑着一个oracle数据库,两个zone,一个weblogic,运行得好好的.说它不太好维护,其实是不太好上手,其实这个是所有linux和unix操作系统的通病.它们是面向技术人员的,而非面向普通用户,因此一些配置操作不是很友好.另外solaris的参考资料相对来说不是很健全.碰到一些问题时,去网上找,不一定就能快速找到答案. 
    磕磕碰碰地,自己看solaris管理员指南,上网查资料,请教公司老员工,最后慢慢也熟悉了在这个操作系统上安装软件,安装与维护oracle数据库,管理用户等日常工作. 
    09年刚开始的时候,网上有消息说sun要被oracle收购.其实我心里是很伤感的.sun是一家极富有创新精神和奉献精神的公司,但最后还是在市场经济环境中倒下了.oracle收购sun之后,我一度怀疑oracle是否还会继续支持solaris.因为先前sun是打算把solaris开源,以开源的opensolaris来反哺solaris,弥补solaris中的不足的. 
    先是opensolaris被oracle遗弃(现在去网上搜索opensolaris,还是0906版本的),后面我从网上看了很多oracle相关报道,发现oracle根本就是个闭源的铁杆支持者.而且还是比较喜欢捞钱的.所以担心Solaris后面是否还会再出版本. 
    不过还好,今年年后没多久,偶然在oracle官网上发现Solaris 11 express版本出来了.系统的图形界面从jds变为gnome,友好了很多.性能计数器功能增强.支持的硬件更多.还有一些其他改进.还好,看起来oracle总算没完全废掉Solaris. 
    现在Solaris 11 正式发布了.看了下总述,其中提到Solaris11 集成了灵活的云管理和云支持技术.其它的倒是在先前版本中能看到.所以我估计oracle将云这个概念也引入到了Solaris中.看样子IT行业已经走进云时代了. 
    虽然我还没去试用,但是我对这个新版本还是有信心.后面我会去安装Solaris 11试用一段时间.

 

后记,刚才去看了下Solaris官方博客(https://blogs.oracle.com/solaris/entry/oracle_solaris_11_the_first),里面写到

Oracle Solaris 11: The First Cloud OS

看了半天,就说了下Solaris提供了ubuntu软件仓库的类似功能.反正就是感觉oracle过于吹嘘Solaris 11的新特性了.
