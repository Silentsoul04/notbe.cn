Convert Access files to Pocket Access using Activesync 4.2 on windows mobile 5.0

Nov 11, 2017

                   

    在06年的时候，我在Embedded Visual C++4.0的开发环境下，写了一个利用ADOCE来访问Pocket Access的程序，具体可以参考这个链接：

      http://www.winbile.net/bbs/forums/threads/1002021.aspx.

    当时我用的是Pocket PC 2003se的机子，通过Activesync，可以将PC机上的.mdb文件转换成Pocket PC上的.cdb文件。

    Windows Mobile 5的系统出来以后，拿到了一个机子做调试，照例用Activesync与PC同步，复制.mdb文件，但是粘贴到PPC上去时，弹出一个对话框说远程调用失败(Remote Procedure Call failed)。上网一查，说WM5已经放弃了对ADOCE的支持：http://www.geekzone.co.nz/forums.asp?ForumId=18&TopicId=5365. 在gogole网上论坛的”Pocket Access and .NET CF 2.0”中，MikeB说，在WM5上，CeDB被eDB替代了，而Pocket Access是基于CeDB的。微软现在还没有把对ADOCE的支持做上去。

    当时一看到这个有点懵，这就意味着我必须要重新coding了，转到SQLCE上去。只有这样么，真是有点不死心，有没有简单一点的方法啊。

    在Google上找了一段时间，发现了一个解决方法，挺不错的，可以偷个懒。具体可以参考这个帖子”convert access files to pocket access using activesync 4.2 and windows mobile 5.0”,帖子上说，只要安装一个插件就可以。看到这个，不由的高兴了一把，下载过来试用了一下，果然可以。

    希望这个帖子对仍旧在WM5上使用Pocket Access数据库的朋友有用。我把文件下载过来了，有需要的朋友可以在这里下：

    http://files.cnblogs.com/dearsj001/ADOCE.WM5.rar.

   

         Good Luck！
