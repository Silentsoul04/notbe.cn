git的基本使用

May 12, 2019

使用git前的SSH配置：

  • 配置SSH
    git config --global user.name "用户名"
    git config --global user.email "邮箱"
    注意：户名和邮箱是git提交代码时用来显示身份和联系方式的，并不非得是github用户名和邮箱
  • 生成秘钥
    ssh-keygen -t rsa -C "邮箱"
  • 设置公钥
    打开id_rsa.pub文件，把内容拷贝到github
  • 验证公钥（非必须进行）
    ssh -T git@github.com

使用git上传代码到GitHub：

初始化本地仓库： git init
查看当前状态： git status

1 添加
$ git add+文件名.文件类型
也可以一次提交整个文件夹：git add + 文件夹名

也可能很多文件需要添加，一个个添加会很繁琐，所以：
	git add -A 提交所有变化	
	git add -u 提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)	
	git add .  提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件

2 提交
$ git commit -m "修改注释"

3 查看修改
git diff 文件名.类型后缀

4 查看操作日志
git log
git log –pretty=oneline

5 版本回退
git reset –hard HEAD^：回退到上一个版本
git reset –hard 版本号：回退到指定版本
git reflog：获取到版本号

6 连接远程仓库

前提：配置好SSH 并且建立本地仓库

GitHub查看command,例如:
git remote add origin https://github.com/***（用户名）/***（仓库名称）.git

7 push代码
git push -u origin master

8 删除文件
rm 文件名.类型后缀
git rm -r --cached fileName

注：文件大小有上传限制
git config --global https.postBuffer 1048576000 ## 设置https缓存为1000M
