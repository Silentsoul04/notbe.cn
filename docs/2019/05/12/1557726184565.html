1.什么是Rancher

Rancher是一个开源的企业级全栈化容器部署及管理平台。Rancher为容器提供一揽子基础架构服务：CNI兼容的网络服务、存储服务、主机管理、负载均衡、防护墙……Rancher让上述服务跨越公有云、私有云、虚拟机、物理机环境运行，真正实现一键式应用部署和管理。

Rancher官网

2.Rancher安装

2.1 安装前准备

  1. 可上网的linux主机(当宿主机使用，需要跑很多容器，如果是生产环境，内存要求尽量大)
  2. 安装Docker，为了速度设置ustc镜像（不懂可以看本人前面的博客）

2.2 Rancher安装启动

##1.下载Rancher镜像
docker pull rancher/server
##创建Rancher容器（9090为宿主机映射的端口号）
docker run ‐di ‐‐name=rancher ‐p 9090:8080 rancher/server

3.Rancher使用

安装完成后，浏览器输入宿主机IP+端口就可以进入欢迎界面了

3.1添加环境

(1)选择“Default -->环境管理” 菜单


（2）填写名称，点击“创建”按钮

（3）按照上述步骤，添加测试环境和生产环境

3.2添加主机

（1）选择基础架构–>主机 菜单，点击添加主机
（2）拷贝脚本

（3）复制脚本后在，在宿主机上运行脚本（宿主机需要和rancher服务器网络相同，本例是rancher和镜像宿主机是同一机器）
（4）点击关闭按钮后，会看到界面中显示此主机。我们可以很方便地管理主机的每个容
器的开启和关闭

3.3添加应用

点击应用–>全部(或用户) ，点击“添加应用”按钮
应用 指的是一类应用的分组（如DB、redis、某服务等等）

填写名称和描述，点击“创建”按钮，即可列表中增加了新增的应用

3.4应用部署

镜像：centos/mysql-57-centos7 增加数据库服务

注意：添加环境变量 MYSQL_ROOT_PASSWORD

点击创建按钮，完成创建 上述操作相当于以下docker命令

docker run ‐di ‐‐name mysql ‐p 3306:3306 ‐e MYSQL_ROOT_PASSWORD=root
centos/mysql‐57‐centos7

完成后服务列表中存在并且状态为激活 测试链接，执行建表语句
