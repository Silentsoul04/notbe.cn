git fork团队协作

May 12, 2019

使用场景
当自己没有主仓库的push权限时，则需要fork到自己的远程仓库，修改完成再提merge request到主仓库。

具体流程

  1. 从主仓库fork到自己的远程仓库
  2. 将主仓库pull到本地仓库，默认远程主仓库用origin表示
  3. 添加自己的远程仓库到本地仓库。git remote add #{name} #{my_remote_repo}。其中#{name}表示自己的远程仓库的代号，比如my-origin。而#{my_remote_repo}表示自己的远程仓库的地址。
  4. 每次需要对代码进行修改时，先切换到origin的master分支，拉取origin最新的代码。
  5. 然后切换到一个新分支，比如git checkout -b dev-xxx，在这个分支上开发。
  6. 开发完成之后，切换到origin的master分支，再次拉取origin最新的代码。
  7. 接着切换的步骤5创建的分支，执行变基命令。git rebase master
  8. 解决冲突
  9. 然后push到自己的远程仓库。git push my-origin 仓库地址
  10. 最后在页面上对这个新分支提merge request到主仓库
