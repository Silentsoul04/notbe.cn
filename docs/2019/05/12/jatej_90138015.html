1.集合框架的概述

1.1 集合的由来

      数组长度是固定,当添加的元素超过了数组的长度时需要对数组重新定义,太麻烦,java内部给我们提供了集合类,能存储任意对象,长度是可以改变的,随着元素的增加而增加,随着元素的减少而减少 。集合是JAVA中提供的一种容器，可以用来存储多个数据。 集合是JAVA中提供的一种容器，可以用来存储多个数据。Java集合框架(java Collection Framework，简称JCF)主要功能是用来将存储的数据以某种结构组织，并以特定的方式来访问这些数据，其目标是提供一个处理对象集合的通用框架，减少程序员处理不同对象集合时的编码量。

1.2 数组与集合的区别

 1）数组既可以存储基本数据类型,又可以存储引用数据类型,基本数据类型存储的是值,引用数据类型存储的是地址值；集合只能存储引用数据类型(对象)集合中也可以存储基本数据类型,但是在存储的时候会自动装箱变成对象

2）数组长度是固定的,不能自动增长；集合的长度的是可变的,可以根据元素的增加而增长；

3）数组中存储的是同一基本类型的元素；集合中存储的都是对象，而且对象的类型可以不一致；

1.3 数组与集合的使用

   元素个数固定并且类型相同，用数组；元素个数不固定的时候用集合，大多数情况下使用集合。在能使用数组的情况下推荐使用数组，因为部分集合的底层是采用不断创建新的数组的方式形成的，不断重复创建数组会会占用大量的内存空间。

1.4 集合框架体系及使用

  集合按照存储方式可以分为两大类，分别为单列集合（java.util.Collection）和双列集合（java.util.Map）.其中，Collection是单列集合类的根接口，用以存放一系列符合某种规则的元素，有List和Set两个重要的子接口。其中Lis接口t的缺点是元素有序，元素可重复，而Set接口的特点是元素无序，元素不可重复。

   List接口下面拥有三种大的常见实现类的特点:

        其中ArrayList底层是通过数组实现，LinkedList底层是通过链表算法实现，Vector底层是通过数组实现

Set实现类特点：

      其中TreeSet集合和HashSet集合存放元素无序，存放和取出元素不一致，而HashSet的一个子类集合LinkedHashSet集合是有序的集合

List接口实现类和Set接口实现类的特点：

1）List集合允许存放有序的元素，而Set集合不允许存放有序元素；

2）list集合允许存放重复的元素，而Set集合不能存放重复元素；

3）List集合有索引，可以使用普通的for循环进行遍历，而Set集合没有索引，不能使用普通的for循环进行遍历；

2.Collection接口

   java.util.Collection接口是所有单列集合的最顶层的接口，里面定义了所有单列集合共性的方法，所有单列集合都可以使用Collection接口中定义的方法。

2.1 Collection中常用功能

 1）public boolean add(E e) 把指定元素添加到集合中去

2）boolean remove(Object o) 把给定对象在当前集合中删除
3） void clear() 清空集合中所有元素
4）boolean contains(Object o) 判断当前集合中是否为给定对象
5） boolean isEmpty() 判断当前集合是否为空
6） int size() 返回集合中元素的个数

7） public Object [] toArray() 把集合中的元素存储到数组中

2.2 迭代器

       在程序开发中，经常需要遍历集合中所有元素，针对此类需求，JDK专门提供了一个接口 java.util.Iterator.Iterator接口也是java集合中的一员，但是与Collection和Map不同：Collection接口和Map接口主要用于存储元素，而Iterator主要用于迭代访问（遍历）Collection中的元素，因此Iterator对象也被成为迭代器。

2.2.1 获取迭代器的方法：

       Iterator是一个接口，无法直接使用，需要使用Iterator接口的实现类对象，获取实现类的方式比较特殊。Collection接口中有一个方法，叫iterator(),这个方法返回的就是迭代器的实现类对象

public Interator Iterator() 获取集合对应的迭代器，用来遍历集合中的元素

2.2.2 Iterator接口的常用方法：

 pulbic E next(); 返回迭代的下一个元素

public boolean hasNext() 如果仍有元素可以迭代，则返回true

2.2.3 迭代器使用步骤

1）使用集合中的方法iterator()获取迭代器的实现类对象，使用Iterator接口接收（多态）；

2）使用Itertor接口中的hasNext()方法判断是否还有下一个元素；

3）使用Iterator接口中的方法next（）取出集合中的下一个元素

注意：Iterator<String>接口也是有泛型的，迭代器的泛型跟着集合走，集合是什么泛型，迭代器就是什么泛型
