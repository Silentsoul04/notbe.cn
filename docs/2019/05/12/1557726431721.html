Linux简介

May 12, 2019

一.计算机的组成及其功能

1.计算机的组成部分

1.运算器：实现算术运算和逻辑运算（cpu组成之一）。
.2.控制器：将指令逐条从存储器中取出，经译码分析后向全机发出相应的操作控制信号作用于其他部件，使各部件协调工作（cpu组成之一）。
3.存储器：将输入设备接收到的信息以二进制的数据形式存到存储器中（有内存，外存之分）。
4.输入设备：将数据，程序，文字符号，图像，声音等信息输送到计算机中。
5.输出设备：将计算机的运算结果或者中间结果打印或显示出来。

二.内核功能及其作用

1.内核的作用

1.作用：计算机系统主要由硬件构成，然后核心程序主要用在管理硬件，提供合理的计算机系统资源分配（包括CPU资源，内存使用资源等）。

2.内核的特点：
1）核心是参考硬件规格携程的，所以同一个操作系统程序不能在不一样的硬件架构下运行。

2）内核仅是在管理整个硬件资源，包括CPU，内存，输入输出装置及文件系统文件。

3）应用程序的开发都是参考操作系统内核提供的开发接口，所有应用程序只能在该操作系统上面运行，不能在其他操作系统上面运行。

2.内核的核心功能

1.系统调用接口（System call interface）：为了方便程序开发者与核心进行沟通，将硬件的资源进一步利用，提供的相关接口。

2.程序管理（Process control）：在多任务的环境下，提供了让核心能够控制多任务，让CPU的资源更有效地分配的机制。

3.内存管理（Memory management）：用于控制整个系统的内存管理，并且kernel会提供虚拟内存功能，当内存不足时可以提供内存置换（swap）功能。

4.文件系统管理（Filesystem management）：文件系统的管理，例如数据的I/O，不同文件格式的支持等。

5.驱动程序（Device drivers）：用于管理各硬件。

6.网络功能（Network features）：用于控制相关的网络活动。

三.Linux常见发行版

1.Debian

1.Debian系列：包括Debian和Ubuntu，以及Linux Mint，Debian是迄今为止最遵循GNU规范的linux系统。Debian分为三个版本分支：stable，testing，unstable。

2.Ubuntu：基于Debian的unstable版本，其中包含最新的软件包，但是业有相对较多的bug，适合桌面用户。

3.Linux Mint：基于Ubuntu的发行版
。
4.Debian系列的包管理使用apt-get/dpkg的方式。

2.Redhat

1.Redhat系列:：包括RHEL（Redhat Enterprise Linux，收费版），Fedora Core（由原来的Redhat桌面版发展而来，免费版本），Centos（RHEL的社区克隆版本，免费）等。

2.Fedora ：由Red Hat Linux个人版衍生出来的免费发行版，每6个月发行一个新版本。

3.RHEL：Redhat Enterprise Linux，也就是所谓的Redhat Advance Server，收费版本

4.Centos：兼容RHEL的格式，免费版，代码全部基于RHEL源代码进行重新编译。

5.Fedora Core：由原来的Redhat桌面版发展而来，免费版本。

6.Redhat系列的包管理使用yum/rpm的方式。

3.SUSE

1.SUSE Linux系列：简称SuSE（Software and system development）。

2.SLES（企业版）：SUSE Linux Enterprise Server

3.openSUSE：现在的 SUSE Linux由openSUSE 项目所维护。

4.Gentoo

1.Gentoo：Gentoo是linux世界最年轻的发行版本，吸取了在它之前的所有发行版的优点，这也是Gentoo 被称为最完美的Linux发行版本的原因之一。并且使用时需要对系统进行编译，安装使用较困难，可通过它了解linux系统内部的运作原理。可根据需求进行优化或定制。

5.Arch Linux

1.ArchLinux：是一个为那些想了解Linux一切的人准备的发行版，因为必须手动安装一切，它会让你学会基于Linux的操作系统的每个部分。

ps：下图为来自http://i.linuxtoy.org/docs/guide/ch48s09.htmlLinux发行版分支图，列举了从1991年到2007年间各种Linux发行版的发展和它们之间的关系。

6.各发行版的标准

1.为了让所有的发行版开发不至于差异过大，且让开发商在开发时有所依据，有Linux Standard Base（LSB）标准来规范开发者，以及目录架构File system Hierarchy Standard（FHS）标准进行规范。
2.其唯一差别，在于各发行版自家开发出来的管理工具，以及套件管理模式的不同。

四.常见开源协议

1.Apache License

1.这是一个著名的非盈利开源组织Apache才用的协议，它鼓励代码共享和尊重原作者的著作权，同时也允许代码修改，再发布（作为开源或商业软件）
1）要求：

1.在代码中保留作者提供的协议和版权信息。
2.如果修改了代码，则必须在被修改的文件中进行说明。

2）允许的权利：

商用，分发，修改，专利授权，私用，附加协议

3)禁止项：

1.禁止因使用等造成影响，承担责任。
2.不能使用相应的商标。

ps：商业软件可以使用，也可以修改使用Apache协议的代码。

2.GPLv3

1.此协议为最广泛的开源协议，有较强的版权自有要求，也赋予和保证了开源项目开发者广泛的权力。它允许用户合法复制，分发和修改软件，但衍生的代码分发也需要开源并遵守此协议。
1）要求：

1.修改后的源码也需要公开。
2.版权及协议也要于此协议一致。
3.修改后，需要在相应的文件做说明。

2）允许的权利：

商用，分发，修改，专利授权，私用

3)禁止项：

1.禁止因使用等造成影响，承担责任。
2.禁止在软件分发传播过程中附加上原来没有的协议条款等。

ps：商业软件不能使用GPL协议的代码。

3.LGPL

1.其主要用于一些代码库，LGPL比起GPL授予的权限较少，LGPL允许商业软件通过类似库引用方式使用LGPL类库而不需要开源商业软件的代码。因此使用LGPL协议的开源代码，可以被商业软件作为类库引用并发布和销售。但是如果修改了源代码的话也必须使用LGPL协议贡献源码出来。
1）要求：

1.公开使用了LGPL部分的代码，其余部分不需要公开。
2.可以库引用的方式用于商业软件。
3.在代码中保留作者提供的协议和版权信息。

2）允许的权利：

商用、分发、修改、专利授权、私用、附加协议

3)禁止项：

1.禁止承担责任，(免责申明)

ps：商业软件可以使用，但不能修改LGPL协议的代码。GPL/LGPL都保障原作者的知识产权，避免有人利用开源代码复制并开发类似的产品。

4.BSD

1.BSD开源协议是一个给予使用着很大自由的协议。基本上使用者可以自由的使用，修改源代码，也可以将修改后的代码作为开源或者专有软件再发布。与MIT协议中只存在细微的差异。差别为MIT可以使用原名称进行宣传，而BSD不可以。
1）要求：

1.在代码中保留作者提供的协议和版权信息

2）允许的权利：

商用、分发、修改、私用、附加协议

3)禁止项：

1.禁止承担责任，(免责申明)

ps：商业软件可以使用，也可以修改使用BSD协议的代码。

5.MIT

1.宽松简单且精要的一个协议。在适当标明来源及免责的情况下，它允许你对代码进行任何形式的使用,也就是原作者只保留版权,而无任何其他了限制,而你必须在发行版里包含原许可协议的声明,无论你是以二进制发布的还是以源代码发布的。
1）要求：

1.在代码中保留作者提供的协议和版权信息

2）允许的权利：

商用、分发、修改、私用、附加协议

3)禁止项：

1.禁止承担责任，(免责申明)

ps：商业软件可以使用，也可以修改MIT协议的代码，甚至可以出售MIT协议的代码。

五.linux的哲学思想

1.linux中一切都是文件：把几乎所有资源统统抽象为文件形式，包括硬件设备，甚至通信接口等。

2.由众多目的单一的应用程序组成：即为一个程序只做好一件事情，组合小程序完成复杂任务。

3.尽量避免跟用户进行交互：易于以编程的方式实现自动化任务。

4.使用文本文件保存配置信息

六.Linux根目录结构

1./boot：系统引导启动时需要加载的静态文件，内核，ramdisk及grub等。

2./bin：系统启动或运行时可能会用到的核心二进制程序，一般用户及root皆可使用的命令。

3./sbin：系统开机过程中所使用到的命令，开机，修复，还原系统所用到的命令，例如：fdisk,fsck,ifconfig,mkfs。

4./lib：开机用到的或/bin,/sbin指令使用时需要呼叫的共享库。
1）libc.so*动态连接的C库

2）ld*运行时链接器/加载器

3）/lib/modules存储内核加载模块（驱动程序）

5./lib64：用于x86_64系统上的辅助共享库存放位置。

6./etc：大多数系统配置文件的存放目录。
1）/etc/opt存放/opt文件的相关配置文件（必要）

2）/etc/X11/存放X window的相关配置文件

3）/etc/sgml存放SGML相关配置文件

4）/etc/xml于XML相关配置文件

7./usr（UNIX Software Resource）：UNIX操作系统软件资源存放位置
1）/bin：所有一般用户能使用的命令。

2）/sbin：与/sbin作用相同

3）/lib

4）/lib64

5）/local：第三方应用程序的安装目录，也存在
bin.sbin,lib,lib64,etc,share

6）/share：主要放置只读文件，共享文件，/man帮助文件，/doc软件文件说明，/zoneinfo时区有关的文件

7）/includeC程序的头文件

8）/libexec某些不被一般使用者惯用的执行档或脚本

8./var：经常发生变化的文件存放位置。
1）/cache应用程序缓存数据目录

2）/lib应用程序状态信息数据

3）/local为/usr/local下应用程序存储可变数据

4）/lock锁文件

5）/log登录文件

6）/opt为/opt下的应用程序存储可变数据

7）/run程序或服务启动后，将PID放入此目录

8）/spool放一些队列数据

9）/tmp保存系统两次重启之间产生的临时数据

10）/mail电子邮箱目录

9./mnt：临时文件系统挂载点。

10./media：便携式设备的挂载点。

11./proc：输入内核及进程信息相关的虚拟文件系统，由内核参数映射而来。

12./sys：输出当前系统上的硬件设备相关信息的虚拟文件系统，内核中与硬件相关的信息映射。

13./root：管理员家目录。

14./home：普通用户的家目录。

15./dev：所有设备文件及特殊文件的存储位置。

16./opt：第三方应用程序安装目录.

17./srv：服务类程序用到的中间数据。

18./tmp：临时文件存放位置。
