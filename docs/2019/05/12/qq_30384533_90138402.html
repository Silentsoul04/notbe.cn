ubuntu pycharm环境

May 12, 2019

1.为什么要配置python虚拟环境？

   virtualenv为应用提供了隔离的Python运行环境，解决了不同应用间多版本的冲突问题。在pycharm中新建项目选择Virtualenv，这样我们就得到了一个不带任何第三方包的“干净”的Python虚拟环境，已经安装到系统Python环境中的所有第三方包都不会复制过来。
   例如：在开发Python应用程序的时候，系统安装的Python3只有一个版本：3.4。所有第三方的包都会被pip安装到Python3的site-packages目录下。如果我们要同时开发多个应用程序，那这些应用程序都会共用一个Python，就是安装在系统的Python 3。如果应用A需要Python2.7，而应用B需要Python3.4怎么办？
   这种情况下，每个应用可能需要各自拥有一套“独立”的Python运行环境。Virtualenv就是用来为一个应用创建一套“隔离”的Python虚拟环境。

2.在pycharm建立工程的时候选择 virtualenv ，报错：

ubuntu 下 ModuleNotFoundError: No module named ‘distutils.core’
解决：

$ sudo apt-get install python3-distutils

windows下

$ python -m pip install -U pip setuptools virtualenv
