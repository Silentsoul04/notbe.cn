Variable used in lambda expression should be final or effectively final

今天打代码遇到一个问题，不知道大家在写代码的时候有没有遇到过：
使用循环创建多个子线程的时候出现下面的问题：

int i = myFolder.getLoadedImagesNum();
for(int k=0; k<4; ++k) {
    executorService.submit(
	    () -> {
		    for (int start = i+k, len = myFolder.getChildImageNumber(); start < len; start += 4) {
		    	//就是在上面这一行
		    	//其中的start=i+k，就会报上面这个错
		    	myFolder.getChildImages().add(new MyImage(files[start]));
		    }
	    });
    myFolder.finishLoad();
}

报错显示要求k是final的，我理解的是为了避免匿名子线程开始了之后，还没有执行到start=i+k之前，父线程就把k的值改变了，这时候得到的结果就和我们预期的不一样了，而我们又需要不断改变k的值，解决方法其实很简单，就是要使得子线程里用到的值都不要发生变化嘛，再定义一个变量，将i+k的值赋给它，然后再线程里用到这个变量就行了：

int i = myFolder.getLoadedImagesNum();
   for(int k=0; k<4; ++k) {
        int j = i+k;
        executorService.submit(
              () -> {
                 for (int start = j, len = myFolder.getChildImageNumber(); start < len; start += 4) {
                    myFolder.getChildImages().add(new MyImage(files[start]));
                 }
              });
        myFolder.finishLoad();
  }

现在就没有报错了。
