适用于C和C ++的跨平台IDE——“JetBrains CLion”

May 12, 2019

JetBrains CLion 2019是一款适用于C和C ++的跨平台IDE，功能强大的智能编码辅助和代码分析软件，使用 CLion能够通过即时导航和可靠的重构来提升你的工作效率，强大的智能代码辅助，让你省时省力又省心！现在JetBrains CLion 2019更新了！现在它的C ++重构现在更加准确，代码突出显示已转移到Clangd以使编辑器更具响应性。为了帮助您遵循首选的编码风格，CLion现在与着名的ClangFormat工具集成，并增加了对各种C / C ++命名方案的支持。对于编译数据库项目，现在可以创建独立于构建系统的目标和运行/调试配置。有需要的朋友，欢迎点击进入macdown下载体验！

JetBrAIns CLion 2019新增功能

嵌入式开发
STM32CubeMX集成
如果您开发STMicroelectronics主板，您会很高兴知道CLion现在与STM32CubeMX集成：
CLion可以创建一个新的.ioc项目或打开一个现有项目，并生成一个CMake文件以正确使用它。
您可以随时直接从CLion启动STM32CubeMX以更新电路板设置并生成代码。
CLion提醒您选择一个电路板配置，并在UI中建议一个选项列表。

OpenOCD支持
现在，CLion支持使用Open On-Chip调试器进行调试。使用新的运行/调试配置模板“OpenOCD Download and Run”在您的微控制器上进行调试。
如果您在CLion中使用STM32CubeMX项目，将自动为您创建OpenOCD运行/调试配置。

ClangFormat
ClangFormat工具广泛用于C / C ++世界，甚至被许多开发人员视为标准。它的配置文件可以在许多项目中找到，特别是在开源软件中。它现在在CLion中得到支持！
ClangFormat跨所有IDE操作
CLion 2019.1添加了ClangFormat作为替代代码格式化程序。为您当前的项目或CLion中的所有项目启用它，让CLion使用它来格式化您的代码：
输入编辑器或粘贴代码
调用重新格式代码操作
重新格式化为预提交挂钩
生成样板代码
重构代码
应用快速修复或意图操作使用.clang格式配置
CLion 在您的项目中检测.clang格式的配置文件，并建议切换到ClangFormat。如果有多个配置文件，则可以通过从工具栏切换器中选择“查看<文件名>的ClangFormat选项”来识别在编辑器中打开的特定文件中使用的文件。
没有配置文件？没问题 - CLion将默认为LLVM样式。

命名约定
在CLion中设置命名
CLion现在尊重您提供的命名方案。在以下情况下将使用所选设置：
自动完成代码
生成代码
重构代码
应用快速修复
您可以手动配置风格设置 | 编辑| 代码风格| C / C ++ | 命名约定或从预定义方案中选择命名。命名检查不一致
CLion可以帮助在编辑器中强制执行所选的命名约定。只需启用Inconsistent Naming检查，CLion将突出显示有问题的名称，并建议快速修复以更准确的名称替换所有用法。

C ++支持
使用Clangd进行代码突出显示和快速修复
为了提高编辑器性能，现在可以在基于Clangd的互补语言引擎上实现代码突出显示和快速修复（默认情况下与引擎一起启用）。
对于快速修复，快速修复的位置在Clangd中计算，而修复本身由CLion提供。由Clang提供的错误消息
一段时间以来，我们一直在利用基于Clangd的语言引擎在编辑器中注释代码错误和警告。在v2019.1中，CLion得到了改进，并且由Clang提供了更详细的错误消息。
一个实用的例子是在C ++代码中调试失败的重载决策。改进了重命名重构
我们改进了文件的重命名重构：
当您执行文件重命名时，CLion会询问您是否还要重命名关联的类/结构，反之亦然。
重命名头文件时，相应地更新标头保护。改进的Extract重构
Extract重构现在更加准确，并且：
尊重类型别名，不扩展静态成员
尊重std::命名空间限定符
尊重模板专业化和模板参数成员函数可以是静态的
此新检查报告可以标记为静态的成员函数。它在基于Clangd的语言引擎之上工作，并分析方法体，试图检查它是否依赖this。

调试器
静音变量
为了获得更好的步进性能，或者只需要偶尔观察变量，您现在可以显式地静音变量。通用的每个IDE开关将停止变量计算，变量视图中的每个变量都会获得一个 Load选项，您可以根据需要应用它。反汇编视图
除了GDB后端之外，现在还可以为LLDB提供反汇编视图。我们一般也改进了视图，现在它显示了按功能划分的信息。
和以前一样，在没有源代码的函数上使用Force Step Into（Shift + Alt + F7）时，反汇编视图可以正常工作。
记忆视图
内存视图在调试模式下转储指针后面的内存。要调用它，请在调试器的变量视图中选择一个指针变量，然后按Ctrl + Enter。
在步进期间，如果您打开了内存视图，CLion将突出显示所选内存块中发生的内存更改。自定义构建和运行
自定义构建目标
独立于构建系统的构建目标（设置 |构建，执行，部署|自定义构建目标）允许您添加自定义目标并配置用于构建/清理目标的任何工具。
最典型的用例是编译数据库项目，它缺少有关构建整个项目的信息。自定义运行/调试配置
要从CLion运行和调试自定义目标，请使用自定义生成应用程序模板（运行|编辑配置...）创建相应的运行/调试配置。
调试器将从Custom Build Target的ToolchAIn设置中获取。注入的语言
现在可以相应地处理带有其他语言代码片段的字符串文字。要暂时注入语言，请按Alt + Enter，应用意图“注入语言或参考”，然后选择适当的语言。例如，SQL字符串，HTML代码或正则表达式。根据所选语言，CLion将提供代码突出显示和特定于语言的意图和行动。自定义颜色主题
照亮您的IDE
厌倦了克里昂的黑白主题？从图标到单选按钮和箭头，确切地调整IDE中的每种颜色。
每个主题实际上都是一个主题插件，您可以直接从我们的插件库下载。或者，如果您愿意，可以使用DevKit创建自己的主题插件，并通过JSON文件修改CLion的外观。
最近的位置
最近的位置弹出窗口（Shift + Ctrl + E）按时间顺序列出您访问过的所有位置，以便您可以轻松返回并查找内容。
使用即时搜索按文件名或包含的代码段过滤位置。或者，切换到仅显示更改的位置。

JetBrAIns CLion 2019特色

智能编码辅助
当您专注于重要的事情时，相信CLion会照顾日常事务。通过直观了解您的代码，它可以通过智能和相关的代码完成，即时导航和可靠的重构来提高您的工作效率。
轻松入手
在CLion中启动新项目非常容易，只需单击一下即可将文件添加到项目中。CLion使用CMake，Gradle和编译数据库项目模型。如果您使用其他类型的项目，IDE将帮助您导入到CMake。智能编辑器
使用可分析上下文并了解项目的IDE，您可以比您想象的更快地编写代码。通过代码洞察力尝试智能完成，格式化和有用的视图。导航和搜索
通过代码找到您的方式，即时导航到符号，类或文件。检查调用或类型层次结构，轻松搜索几乎所有内容（包括IDE设置）。代码生成和重构
在CLion为您生成代码时节省不必要的打字时间：从getter / setter到更复杂的模板。使用重构以思考的速度改进和清理代码。即时代码分析
用CLion写出漂亮而正确的代码。所有支持语言的静态分析（包括DFA）会在您键入时立即突出显示代码中的警告和错误，并建议快速修复。编辑器定制
选择一个默认编辑器主题或自定义它们以符合您的个人首选项。采用以键盘为中心的方法，选择您最喜欢的键盘图，或轻松创建自己的键盘图。内置工具和集成
CLion不仅仅是一名编辑，因为它提供了强大的调试器和动态分析工具，可以轻松调查和解决问题，内置Google Test，Boost.Test和Catch进行单元测试，许多流行的VCS支持开箱即用等等。
运行和调试
在本地或远程在CLion中构建，运行和调试应用程序和单元测试。
使用带有GDB或LLDB的调试器UI作为后端。动态分析
使用Valgrind Memcheck和Google Sanitizers集成来检测内存错误，数据争用和未定义的行为问题。使用CPU Profiler集成分析应用程序的性能。CMake支持
CMake是一个着名的跨平台构建系统，广泛用于C和C ++项目。受益于CLion的智能CMake支持，包括代码生成，完成和自动目标更新。
单元测试
CLion支持Google Test，Boost.Test和Catch框架，并提供内置的测试运行器以及强大的UI来调查测试结果。它甚至可以在您编写测试时为您生成代码（对于Google Test）。代码文档
使用CLion可以轻松记录您的代码。在快速文档弹出窗口中预览Doxygen风格的文档，使用命令和函数参数的完成，并为Doxygen注释生成存根。VCS集成和本地历史
CLion为大多数流行的VCS提供了统一的界面，包括Subversion，Git，GitHub，Mercurial，CVS，Perforce和TFS。同时，当地历史将为您免除意外事故。终端，Vim模式，数据库等
通过内置终端访问命令行，将Vim仿真模式与Vim键绑定一起打开，或者使用其他各种插件扩展IDE的功能。
