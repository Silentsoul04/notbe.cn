NEXT:float初衷？包裹？崩溃？

破坏性（也叫崩溃，崩塌）：将子元素设置为浮动之后，父元素在没有指定高度的情况下，
父元素的高度也将会不存在。这个特点称之为崩溃或者崩塌。

包裹特性：在元素在一行显示时，其底部一般都不是紧挨着下边线显示的，而是有一条基准线，基准线离行高的下边线存在一定的距离，很少有元素会超过基准线显示，这就是元素的包裹特性。
如果去除行级元素在显示时产生的漏边现象呢？将行级元素设置为垂直对齐即可解决：
也就是将行级标签元素的垂直项设置为对齐：
img{
vertiacl-align:bottom; //垂直对齐设置为bottom，也就是将行级元素转换为块级元素。
}

CSS定位：
position: 属性值默认为static，即无定位。

相对定位：relative:相对定位的便宜参考元素本身，不会是元素脱离文档流。元素的初始位置占据的空间会被保留。即可能会覆盖掉一些标准流的文件对象。
主要代码：position:relative; //相对定位的参照物是参照它本身.
主要参数有两个：top:xxpx ; left:xxpx ; 分别设置离顶部和左边的距离。
绝对定位：相对于已定位的最近的祖先元素，如果没有已定位的最近的祖先元素，那么它的位置就相对于最初包含块（如body）
主要代码：position:absolute; //绝对定位与相对定位有本质的区别。
块级元素一旦设置了绝对定位后，其本身就变成了浮动状态从而脱离了（标准）文档流，它后面的标准流文件就会顶替它的位置出现。

在块级元素中设置了top和left属性后，必须要设置position的属性值，如果不设定，那么top和left的属性值都不会生效。
4-1：绝对定位与浮动
1-1：绝对定位与浮动的区别：

a 绝对定位产生的浮动效果会覆盖顶替它位置的标准流文件，而使用float设置的浮动则不会产生覆盖文字的效果，只会将文字环绕块进行输出。换句话说就是没有忽略div浮动所产生位置。

将相对路径和浮动组合使用，使得块级元素既具有浮动的特点，又可以对块对象进行方位的设置。但是有一点需要说明的是，对象进行偏移设定之后的效果可能不产生文字环绕的效果。
z-index的使用：它是决定产生覆盖效果的元素谁在上面，谁在下面的设定。它的相对值越大，则越靠上面。

固定定位：相对于浏览器窗口进行定位。
主要代码：position:fixed;
示例：演示div相对浏览器位置不改变的效果。

定位技术小结：
相对定位的参照物是它自己，而绝对定位的元素是离它最近的父一级元素，相对定位不会破坏文档流，而绝对定位会破坏文档流，两者在一起该如何搭配呢？通常情况下是相对定位做父一级单位，而绝对定位作为子一级定位，相对定位的作为父一级元素的时候，可以不设定top和left的属性。依然保留在原位置。仅仅是供绝对定位有一个参照物。那么这样当作为一个整体进行移动的时候便可以以相对定位的父一级元素进行整体移动。所以呢，相对定位和绝对定位，从这个角度来讲，它们是父子的关系。

定位的参照物是谁？
相对定位的参照物是它自己，偏移之后原位置不会被占用，依然会保留。而绝对定位的参照物则是父一级的元素。通常在子元素进行布局的时候我们使用的是绝对定位。进行父元素的时候我们用的是相对定位。

浮动与定位，分别用在什么情况下？
浮动与绝对定位的组合都是会破坏文档流，而绝对定位变成浮动后，会忽略原来它所占用的位置。而浮动则会绕过它所占用的位置。浮动跟相对定位进行组合，浮动和相对定位组合之后便有了两者特有的特点。既可以产生浮动效果，也可以设置top和left等属性值进行偏移的微调。
