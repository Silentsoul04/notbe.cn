May 12, 2019

第一章 继承
第一节、
继承：主要解决代码重复（成员重复定义）的问题
UML：同一建模语言，主要用图形符号来表示面向对象的一种图形语言
类图三部分：类名、成员属性、成员方法
访问控制修饰符
+： public
- protected
# private
~或者空 default
使用关键之 extends 来实现继承
public class Person{
String name;
int age;
}
public class Teacher extends Person{
float salary;
}

继承的概念：从现有类创建子类，现有类称为父类、超类、或基类。
1.子类继承父类中所有成员(构造器除外)。
Java语言规定，一个类只能继承一个父类，这种限制被称为单继承
继承的优点：可以创建更为特殊的类型
消除代码重复性
利于应用程序的可维护性可扩展性
父类中的成员无论是公有的还是私有的，均被子类继承

— private只是访问权限，但不是继承权限

子类不能对集成的私有成员直接进行访问，可通过继承的公有方法来访问。

第二节
方法的覆盖 — 子类重写父类的方法

要求方法签名必须一致

子类方法的访问控制修饰符范围必须大于等于父类被覆盖方法

被覆盖的父类方法不应是private

方法覆盖：子类重写父类中的方法
– 通过子类对象调用父类中的成员方法时，需要实现与父类方法中的不同功能

public class Person{

	        }

方法覆盖注意：@Overrode 注解：用于检查下面的方法是不是真正遵守了父类的规则，做覆盖检查检查下面的方法是否满足覆盖的
要求

方法覆盖规则：
1.方法签名必须一致
方法名，返回类型，参数列表
2.子类中的访问控制修饰符的范围必须大于父类中被覆盖方法访问修饰控制符的范围
3.被覆盖的成员方法不应该为prvate 编译允许通过
4.子类中覆盖方法的返回类型可以是被覆盖方法的返回类型的本类类型或者是子类类型

第四节、调用父类的构造器

– super() 语句：
子类构造器中通过super()语句调用父类构造器
– 由编译器在编译时自动添加(隐式super())隐藏起来我们看不到
隐含super()的说明：
- 调用父类无参数的构造器
- 借用父类的构造器，来初始化本类继承的属性
- 只有在子类构造器第一行不是this(parameterList);也不是super(parameterList)时，
编译器才会自动添加。
可以使用显式的super(parameterList)来调用父类中带有参数的构造器(参数列表一 一 对应)
– 注意：super()语句在构造器中非注释行的第一行.
this()语句在构造器中非注释行的第一行.
所以在构造器中的第一行要么是this,要么是super.
