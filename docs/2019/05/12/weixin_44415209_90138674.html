git commit 忽略指定文件

May 12, 2019

今天遇到了一个问题，我想提交node的代码，里面是一个day3的文件夹，文件夹里面吗有一个node_modules的配置文件夹，里面是庞大的数据，如果要提交到远端上去的话，需要很长时间。现在就想，有没有一个办法把不必要的文件进行忽略掉，百度了一下，找到了，就是新建.gitignore然后再里面声明要忽略的文件夹或者文件就可以了，不过麻烦又来了，因为是点开头，没有文件名，没办法直接在windows目录下直接创建，必须通过右键Git Bash，按照linux的方式来新建.gitignore文件。具体步骤如下：

1.在需要创建 .gitignore 文件的文件夹

（1）、右键选择Git Bash 进入命令行，进入项目所在目录。

（2）、输入 touch .gitignore 在文件夹就生成了一个“.gitignore”文件。
然后在”.gitignore” 文件里输入你要忽略的文件夹及其文件就可以了。

3.然后用编辑器打开这个文件进行编辑就行了。如果你很牛，熟悉linux下的vi编辑的命令，也可以用命令来写，在这里就不说了额。

之后就可以上传了，如果想知道怎么上传的话，可以看我写的上一篇博客。
