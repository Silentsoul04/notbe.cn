Redhat 7.4 部署keepalived,实现web集群高可用

May 12, 2019

环境准备：web服务器1，地址192.168.4.2，web服务器2，地址192.168.4.3.关闭selinux，关闭防火墙。

1、在web服务器上安装keepalived软件。

yum -y install keepalived

2、web1,与web2，同样操作

[root@node2 ~]# vim /etc/keepalived/keepalived.conf 

配置完毕，我们看看备用服务器的配置差别

定义为备份服务器，优先级稍低

这里请大家注意，配置文件中#vrrp_strict。这一项，必须注释。

最后启动服务。

3、systemctl restart keepalived

4、验证

使用客户端访问，应该都是web1的界面。然后模拟服务器故障，关闭web1的httpd服务。访问不受印象。
