Brupsuite:
Spider:爬行的意思。
Request Headers这个配置选项就是可以模仿用谷歌或者百度的头取爬行.就是模拟其它浏览器的头去爬行其它的浏览器。
Spider Status这个意思就是爬行的状态
Request mad 请求的线程数
Bytes transferred 传输字节数
Requests queued 请求的队列
Forms queued 队列的形式
Spider is running暂停爬行
clear queues清除爬行的数据
uid用户
Number of threads爬行的速度.
Crawier Settings就是个爬行的配置需求选项：
robots.txt文件就是禁止爬行不允许的目录，有些目录是允许给爬行的，当然禁止爬行的目录也是可以爬行出来的
勾上check robots.txt这选项就可以爬取出允许爬行的文件目录 Disallow是禁止，Allow允许
Detect custom "nom found"reponses选项就是不会去爬行不存在的页面，如果不勾上的话就会觉得不存在的页面
也是个正常的页面一起爬行下来，一般勾上的.
Request the roo of all directories选项就是请求所有根目录下的目录
lgnore links to non-text content忽视掉非文本的链接,如果有图片的链接就会忽视掉。
Make a non-parameterised rquest to each dynaminc page选项就是请求非汇集动态页面内容，汇集页面
就是图片和文字都在一起的链接。
Maximun link depth这选项就是爬行目录的深度，有些敏感文件后台登入的目录，如果藏得深的话，就可以
设置这参数，可一般5都ok了。
Application Login对表单配置需求:前提是要能爬行出表单
下的Automatically submit these credentials这个配置选项就是设置账号和密码,如果爬行出表单，就可以
试着登陆，登陆成功到后台的话，就会继续爬行后台的数据。
Spider Engine：这配置选项就是设置线程的配置选项
Number of retries on network failure这选项是配置连接失败后最大重试几次
Pause before rety milliseconds：这个是设置连接失败后时隔几秒请求。
Number of threads这个选项是如果爬行大型网站的时候可以把参数设置大点最大好像是200.把参数设置大的话
可以加快爬行的速度。
