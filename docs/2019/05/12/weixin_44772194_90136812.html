Java面向对象三大特征的概述总结

May 12, 2019

Java面向对象三大特征

封装

​ 封装（encapsulation）是计算机常见的术语，既保留有限的外部接口（interface），隐藏具体实现细节。比如在Linux架构就可以看到Linux操作系统封装了底层硬件的具体细节，只保留了系统调用的接口。用户处在封装的外部，只能通过接口，进行所需的操作。

​ 封装在生活中很常见，比如手电筒，用户及时不看说明书，也可以猜到手电筒的操作：开关和充电。这个手电筒用一个塑料壳将用户不需要接触的内部细节隐藏起来，只保留了两个接口，开关和插头。使用这两个接口，用户就可以使用这个手电筒的功能。如果不进行封装，直接将所有的实现细节直接暴露给用户，用户会对具体实现产生疑惑，所以封装提高了产品的易用性。直接暴露所用的实现细节，这样虽然可以让用户对产品进行自由的操作，但是用户需要承担更大的风险（损坏产品），所以封装也提高了产品的安全性。

​ 在Java中，一个对象内部往往有许多成员（数据成员和方法）。有一些数据成员和方法只可以内部使用，这是我们就可以对对象实现封装。

​ Java中封装的关键字：private

下面我们来封装一个类

class Student{
	private String name;
	private int age;
	private double score;
}

上面代码定义一个学生类，并且封装了三个属性，现在这三个属性只能在本类被调用，所有我们需要提供公开的调用和设置方法，即get和set方法。所以上面的类就需要修改：

class Student{
	private String name;
	private int age;
	private double score;
	public Student() {
		super();
	}
	public Student(String name, int age, double score) {
		super();
		this.name = name;
		this.age = age;
		this.score = score;
	}
	public String getName() {
		return name;
	}
	public void setName(String name) {
		this.name = name;
	}
	public int getAge() {
		return age;
	}
	public void setAge(int age) {
		this.age = age;
	}
	public double getScore() {
		return score;
	}
	public void setScore(double score) {
		this.score = score;
	}
}

提供公开的get和set方法，如果需要修改数据，得到数据，就可以通过调用get和set方法。

继承

​ 生活中的继承：父辈对子女的进行财产的赠与，父辈的赠与 子女的获得。

​ 程序中的继承：类与类之间满足is a关系 才能定义继承关系 是指 子类可以去获得父类中非私有的属性和方法。

​ 继承的关键字：extends和implements

​ extends关键字：在Java中，类的继承是单一继承，一个子类只能拥有一个直接父类，extends只能继承一个类。

​ implements ：使用范围为类继承接口的情况，一个类可以实现多个接口，接口和接口之间用逗号分隔。

类的继承格式：

class 父类{
    
}

class 子类 extends 父类{
    
}

公共父类动物类：

class Animal{
	private String name;
	public Animal(String name) {
		super();
		this.name = name;
	}
	public void eat(){
		System.out.println(name + "正在吃");
	}
	public void sleep(){
		System.out.println(name + "正在睡");
	}
}

定义一狗类

class Dog extends Animal{
	public Dog(String name){
		super(name);
	}
}

定义一个猫类

class Cat extends Animal{
	public Dog(String name){
		super(name);
	}
}

上面Animal类作为一个父类，然后其他的狗，猫等动物继承这个类之后，就可以调用动物类中的属性和方法（私有成员除外），子类就不存在重复的代码，这样就提高了代码的复用性，维护也提高了，代码更加简洁。
Java继承关系的图示：

继承的特性：

1.子类拥有父类的非私有的属性和方法，构造方法不能继承。
2.子类可以拥有自己的属性和方法，对父类的功能进行拓展。
3.子类可以对父类的方法进行覆盖。
4.Java的继承是单继承，可以多级继承。

继承的好处：

1.提高了代码的复用性，不能为了代码的复用性强制定义不符合里氏代换原则原则的继承关系。
2.是多态的前提。

多态

​ 从一定的角度看，封装和继承几乎都是为多态而准备的。

​ 多态的定义：允许不同类的对象对同一消息做出不同的响应，即同一消息可以根据发送对象的不同而采用多种不同的行为方式（父类引用指向子类对象）。

​ 多态的特点（编译看左边，运行看右边）：

​ （1）编译时多态：在编译时，只能调用引用类型所对应类中声明的方法。

​ （2）运行时多态：在多态发生时，运行时调用的是子类覆盖父类后的方法。

编译时例子：

public class Demo{
    public static void main(String[] args) {
        //声明一个动物类的引用 创建一个狗的对象放到动物类的引用
        Animal a = new Dog();
        a.eat();
        a.sleep();
        
        a.shout(); //编译报错 
    } 
}
//定义一个动物类
class Animal{
    public void eat() {
        System.out.println("动物吃");    
    }
    public void sleep() {
        System.out.println("动物睡");  
    }

}
//狗类继承动物
class Dog extends Animal{
    //特有的叫方法
    public void shout() {
        System.out.println("狗叫");
    }
}

运行时例子

public class Demo{
    public static void main(String[] args) {
        //声明一个动物类的引用 创建一个狗的对象放到动物类的引用中
        Animal a = new Dog();
        a.eat(); 
        a.sleep();
    }
}
//定义一个动物类
class Animal{
    public void eat() {
        System.out.println("动物吃");    
    }
    public void sleep() {
        System.out.println("动物睡");  
    }
}
//定义一个狗类继承 
class Dog extends Animal{
    //重写 吃和睡
     public void eat() {
        System.out.println("狗吃骨头"); 
    }
    public void sleep() {
        System.out.println("狗趴着睡");  
    }
    //叫
    public void shout() {
        System.out.println("狗叫");
    }
}



运行结果：狗吃骨头        狗趴着睡

实现多态的技术称为：动态绑定（dynamicbinding），是指定执行期间判断所引用对象的实际类型，根据其实际的类型调用相对应的方法。

多态的作用（消除类型之间的耦合性）：

1.多态用在方法的返回值类型上
2.多态用在方法的形参上
3.多态用在数组中

实现多态的必要条件：

1.存在继承关系
2.有方法的重写
3.父类引用指向子类对象

多态的好处：

1.屏蔽子类之间的差异性
2.增强了方法的通用性
3.利于程序的拓展
