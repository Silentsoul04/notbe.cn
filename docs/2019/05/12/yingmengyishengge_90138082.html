MVC

May 12, 2019

1、controller:负责相应用户，调度model/view 不负责sql
model：操作数据库，不针对数据的合法性（可以在外界直接操作数据库）
view:一般不含逻辑，只负责输出查了以后说 （将操作后的结果显示出来）

/*百度查的：model的出现就是为了把数据层全部控制在model，

mvc 的 c 是拿来衔接model和view的

MVC的核心思想是分离界面(View),逻辑(Controller),数据(Model).

*/

注意：下面两个类需要在init.php中调用

class Model{
	protected $table=null;//model控制的表
	protected $db=null;//引入的mysql对象
	
	public function __construct(){
		$this->db=mysql::getIns();//getIns()是数据库类中的方法
		
	}
}

class TestModel extends Model{
protected $table='test';//数据库的表
public function reg($data){
	$this->db->autoExecute($this->table,$data,'insert');
	//自动加载
	
}
public function select(){
	return $this->db->getAll('select * from table');//查看表的信息
}


}
require('./include/init.php');


$test=new TestModel();
var_dump($test->reg(array('t1'=>'frontuser1','t2'=>'frontuser1')));//在外界将数据插入数据库
