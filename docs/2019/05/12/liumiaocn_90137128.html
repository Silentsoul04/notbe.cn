Groovy基础：4:Groovy的注释与Shebang

May 12, 2019

目录

https://liumiaocn.blog.csdn.net/article/details/90137145

在前一篇的Groovy基础中，引出了#!/usr/bin/env groovy的写法，这篇文章主要讲述一下Groovy中的注释与#!的相关知识。

注释

Groovy的注释沿袭Java的方式，主要分成如下几种类型

单行注释

注释格式： // 注释内容

多行注释

注释格式：/* 注释内容
注释信息
…
*/

GroovyDoc

类似与

注释格式：/** 注释内容

  • 注释信息
  • …
    */

注释示例

liumiaocn:sample liumiao$ cat comment.groovy 
#!/usr/bin/env groovy
/**
 * Groovydoc descrpiton : comment example
 */
//single line comment
println "Hello, Groovy" /* multi-line coment: line1
  comment line2
  comment line3
  comment line4 */

println "Hello," + /* Hello Information */ "Groovy" /* Groovy information */
liumiaocn:sample liumiao$

执行示例

liumiaocn:sample liumiao$ groovy comment.groovy 
Hello, Groovy
Hello,Groovy
liumiaocn:sample liumiao$

Shebang行

写在第一行的如下信息被成为Shebang行，Shebang还有其他的名称，比如sha-bang、hashbang、pound-bang、hash-pling等。实际上sha-bang更容易理解，因为#的发音为sharp，而bang则表示为!，所以sha-bang能够很容易知道其实这就是诸如如下的设定行。

Shebang行：#!/usr/bin/env groovy

shebang行并不是groovy所特有的功能，其实除了groovy，python和ruby等也都有类似的做法。这里与其说其为一种注释，不如理解为指定执行方式的入口，groovy的注释和Java的注释一样，#不是groovy可以接受的注释符。
它实际是UNIX系统中的一种机制，在Unix系统中，包含shebang的文本文件是用于将此文件设定为可执行文件的一种方式，最为大家所熟悉的不是groovy的这行设定，而是如下的sh的设定

#!/bin/sh

基本上在大部分脚本中都会有如上的Shebang行的设定，实际上其指定了当前文件执行所使用的加载器（解释程序），比如上述的脚本会使用/bin/sh来进行执行。关于shebang，有如下的使用惯例和注意事项：

  • shebang行需要为程序的首行
  • #!之后的解释程序需要具有可执行权限
  • 脚本文件本身也需要具有可执行权限，才能在Linux或Unix中使用dot操作符直接操作的可能
  • 脚本语言中中没有shebang行的话，会使用$SHELL去执行该脚本
  • #!指定的解释程序需要使用绝对路径
  • 为了解决绝对路径在不同环境下的不同，可以使用#!/usr/bin/env进行解决
