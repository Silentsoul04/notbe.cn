所以问题就出现在*y[i]*的计算过程中，由于我传入的是整型y，在计算的时候也会保持整型，故0.9就会被取为0，从而看到开头的输出结果，改正方法如下,运算之前将数组转化为浮点型即可：

def sigmod(y):
    y = np.array([float (y1) for y1 in y])
    for i in range(len(y)):
        y[i] = 1.0 / (1.0+np.exp(-y[i]))
    return y

2.矩阵运算

我的Python是这个学期一周速成的，所以很多细节还有待提高。矩阵运算是逃不过的死劫。下面的问题困扰我很久，y1.y2,y3到底有什么区别嘿？：

import
y1 = np.array([1,2])
y2 = np.array([[1],[2]])
y3 = np.array([[1,2,3]])
print(y1)
print(y2)
print(y3)
#输出结果
[1 2]
[[1]
 [2]]
[[1 2 3]]

y2自不必说，可以成列向量，那么y1和y3有什么区别呢？看下面的代码：

y1 = np.array([1,2])
y2 = np.array([[1],[2]])
y3 = np.array([[1,2,3]])
print("-------------y1.T-------------------")
print(y1.T)
print("-------------y2.T-------------------")
print(y2.T)
print("-------------y3.T-------------------")
print(y3.T)
#输出结果
-------------y1.T-------------------
[1 2]
-------------y2.T-------------------
[[1 2]]
-------------y3.T-------------------
[[1]
 [2]
 [3]]

于是我们发现了，y1的转置仍然是自己，进一步的：

y1 = np.array([1,2])
y2 = np.array([[1],[2]])
y3 = np.array([[1,2,3]])
print("-------------y1.shape-------------------")
print(y1.shape)
print("-------------y2.shape-------------------")
print(y2.shape)
print("-------------y3.shape-------------------")
print(y3.shape)
#输出结果
-------------y1.shape-------------------
(2,)
-------------y2.shape-------------------
(2, 1)
-------------y3.shape-------------------
(1, 3)

一般情况下：

[1,2]的shape值(2,)，意思是一维数组，数组中有2个元素。

[[1],[2]]的shape值是(2,1)，意思是一个二维数组，每行有1个元素。

[[1,2]]的shape值是（1，2），意思是一个二维数组，每行有2个元素。
