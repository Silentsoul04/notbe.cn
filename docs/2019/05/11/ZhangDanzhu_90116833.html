【leetcode_1029】两地调度

May 11, 2019

公司计划面试 2N 人。第 i 人飞往 A 市的费用为 costs[i][0]，飞往 B 市的费用为 costs[i][1]。

返回将每个人都飞到某座城市的最低费用，要求每个城市都有 N 人抵达。

示例：

输入：[[10,20],[30,200],[400,50],[30,20]] 输出：110 解释： 第一个人去 A 市，费用为 10。
第二个人去 A 市，费用为 30。 第三个人去 B 市，费用为 50。 第四个人去 B 市，费用为 20。

最低总费用为 10 + 30 + 50 + 20 = 110，每个城市都有一半的人在面试。

提示：

1 <= costs.length <= 100 costs.length 为偶数 1 <= costs[i][0],
costs[i][1] <= 1000

https://leetcode-cn.com/problems/two-city-scheduling/

思路

每一个人必然需要去两地之一，我们需要尽量把每个人往费用小的地方送，可以计算两地费用差值（A-B），然后排序，顺次先把去B费用与A差值（含符号）最大的人送往B市，剩下的再去A市。这样才能保证最后的调度总费用最小。

代码

class Solution {
public:
    int twoCitySchedCost(vector<vector<int>>& costs) {
        int cost = 0;
        sort(costs.begin(), costs.end(), 
             [](vector<int> a, vector<int> b){return a[0]-a[1] > b[0]-b[1];}
            );
        for(int i = 0; i < costs.size()/2; ++i) {
            cost += costs[i][1];
        }
        for(int i = costs.size()/2; i < costs.size(); ++i) {
            cost += costs[i][0];
        }
        return cost;
    }
};
