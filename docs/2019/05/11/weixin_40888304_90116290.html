嵌入式系统移植

May 11, 2019

（一）开发环境搭建与ROM烧写。
首先对SecureCRT软件的安装和破解，按照正常步骤安装好SecureCRT软件后，在通过SecureCRT-kg应用程序对其进行注册与破解。接下来安装U转串驱动了，点击对应驱动包进行安装，再安装 SuperVivi-Transfer-ToolComplete驱动，连接TX2440USB口与电脑USB口，启动设备管理器，更新SuperVivi-Transger-ToolComplete驱动。接着将TX2440开发板的USB线和端口线连接到PC从而进行开发板与SecureCRT软件的关联，如果接通开发板电源后，在SecureCRT显示的Linux系统是正常运行，证明成功搭建好了开发环境。最后打开Ubuntu进入虚拟机，通过相关操作完成例子中对zImage代码的烧写。
（二）、嵌入式系统启动过程分析与嵌入式Linux常用命令。
嵌入式Linux系统操作涉及到Linux常用命令、Vi编辑器、gcc编译器、Shell编程等等命令工具的使用，还涉及到Linux系统编程入门相关知识的运用。其中Linux命令的一般格式如：# command [-options] parameter1 …；Vi是最早的一种交互式可视化文本编辑器；gcc是符合ISO标准的C编译器，它在需要的时候调用其它组件（预处理器、汇编器、连接器）；而shell是一种具备特殊功能的可执行程序，它是介于使用者和 UNIX/linux 操作系统内核间的一个接口。
安装U转串驱动。双击PL2303_Prolific_DriverInstaller_v110安装驱动。安装 SuperVivi-Transfer-ToolComplete驱动，连接TX2440USB口与电脑USB口，启动设备管理器，更新SuperVivi-Transger-ToolComplete驱动，将TX2440开发板的USB线和端口线连接到PC。拔下开发板跳线帽，进入下载模式，打开SuperVivi-Transfer-ToolComplete下载软件，重启TX2440开发板，选择[2]下载镜像模式，点击upload选择zImage_2.6.31.bin文件。等到zImage_2.6.31.bin文件刷写完成，切断开发板TX2440电源，装上跳线帽。SecureCRT连接开发板，接通开发板电源，在SecureCRT显示的Linux系统是正常运行。
（三）、嵌入式Linux应用开发与交叉编译。
在一种计算机环境中运行的编译程序，能编译出在另外一种环境下运行的代码，这个编译过程就叫交叉编译。简单地说，就是在一个平台上生成另一个平台上的可执行代码。在进行交叉编译过程中，我们需要在主机平台上安装对应的交叉编译工具链（cross compilation tool chain），然后用这个交叉编译工具链编译我们的源代码，最终生成可在目标平台上运行的代码。而本次见习通过在Ubuntu中编写一个能在SecureCRT上打印出“hello world”内容的简单程序来熟悉如何运用交叉编译。
（四）、嵌入式应用（图像、播放器）移植与调试。
嵌入式linux截图工具的基本原理就是通过读取Framebuffer，把LCD上显示的内容转换成图片 格式文件(jpeg、bmp、png)。 编译截图工具，需要系统的jpeg库（libjpeg.so）的支持，一般交叉编译器中没有jpeg库，需要 自己安装。
在对播放器的移植与调试中，首先准备源码包（本次见习已给出），再在Ubuntu中建立两个文件夹，一个存放源码，一个存放安装文件 ，之后通过编译zlib-1.1.4 、libid3tag、libmad和madplay，从而在madplay-0.15.2b目录下生成madplay的可执行文件。之后将madplay复制到根文件系统/usr/bin目录下，将一个MP3文件也放在某一目录下，在开发板上 执行madplay程序。
